自从大模型及智能体技术兴起以来，我们每个人都在亲身经历这场智能化革命带来的冲击。然而，在这场浪潮中，我也看到不少人有一些迷茫——并非不努力，而是努力之后依然看不清方向。甚至也有不少人，为了不落伍，跟着热点跑上跑下，疲惫不堪，却收效甚微。

## 学会跳出技术看技术

这里我先用一个比喻来说明这个问题：如果说技术的演进像一条大河，那么每一次变革就像一朵浪花。许多人的方式是“跳进水里”与浪花“贴身肉搏”，却也可能因此失去了全局视角。其实，真正看清浪花走向的，往往是那些站在岸边、从远处观察的人。

同样的道理也适用于技术世界，就是我们常说的“跳出技术看技术”。因此，看待智能体，就要跳出智能体本身去理解它；看待中台，也要跳出中台去审视它等等。

因为，就像浪花的形成不仅取决于水流本身，更受到河床、地势等深层因素的影响。同样地，真正驱动这些技术背后的力量，是其背后的思想与理念。

## 技术演进对人类社会的冲击

在课程的结束语，我还想和你聊聊大模型对 IT 真正的冲击是什么？

在回答这个问题之前，我们不妨从一个更宏观的视角出发，看看技术演进对整个人类社会的冲击。

在此前的专栏中，我们多次提到“信息流”的概念，但讨论的视角大多限定在 IT 的范畴。

实际上，整个现实世界的运转本质上就是一个庞大的信息流网络。在这个网络中，企业、机构、人、货物、机器、系统……所有的存在，都是网络中的节点，通过信息的流动和交互，实现整体的协同与运作。

社会学中有一个经典理论，叫做“社会加速理论”。它指出，随着农业时代、工业时代到信息时代的演进，人类社会的信息流动速度在不断加快。

而随着智能化时代的到来，这一趋势将进一步加剧。

推动这一趋势加速的，有两股不容小觑的力量。

一是各种智能节点的注入，典型的如智能体，它们具备感知、决策和执行的能力，成为网络中更主动、更智能的参与者。

二是大模型的普适性，它几乎可以赋能网络中的每一个节点，无论是人还是机器，都可以通过它提升自身的智能水平和处理能力。

在这两股力量共同作用下，会推动整个网络的运行速度继续上升。而“社会加速理论”同样指出，如果一个节点无法跟上整个网络的节奏，不论是人还是机器，就很容易被边缘化，甚至被淘汰，这正是一直以来“进化”的本质。

但比起“被淘汰”这件事情，我们更关注的是，从演进的视角来看，作为网络中的个体，如果想不被淘汰，应当朝着什么方向进行演化和提升？

答案就是：极致的个体 + 极致的协同。

这是一对相辅相成的趋势。“极致的个体”意味着节点自身要变得更强大、更智能；“极致的协同”则意味着节点之间的协作要更高效、更紧密。

这一演化逻辑，适用于社会网络中的任何层级。我们可以通过两个典型例子来进一步说明。

一个例子是企业。企业作为社会网络中的核心节点，正在经历从“有边界”到“无边界”的转变，其协同模式也由传统的线性价值链向更为开放、多元化的价值网络模式转变。

另一个例子是物品。以汽车、手机、家用电器为代表的物理实体，也正在经历从“被动工具”向“智能节点”的跃迁。通过信息化与智能化的注入，它们具备了数据感知、信息处理，甚至自主决策的能力，能够与人、系统、环境进行实时交互与协同运作。

## 大模型对于 IT 的真正冲击

现在，我们再回归到 IT 的范畴，分析一下大模型对于 IT 的真正冲击是什么？

IT 本身是社会信息流网络中的一个子网络。而在这个子网络中，每一个节点也都将经历“极致的个体 + 极致的协同”的演进过程。

事实上，在大模型出现之前，这种趋势已经出现。我们可以借助《教堂与大集市》中的观点来理解一下：传统的软件开发，像“教堂”一样封闭、集中，而开源社区则像“大集市”一样开放、分布。这种“大集市”式的开发模式，在开源领域已经取得了显著成果。

然而，在过去，这种“集市式”的开发理念并未真正深入企业 IT 内部，其核心障碍在于软件开发的高知识门槛，以及业务与 IT 之间的天然壁垒。

而如今，大模型带来的智能编程正在逐步打破这一壁垒。它让非专业开发者也能参与到系统建设中，为“科技 + 业务”共建提供了前所未有的可能。

这种模式，其实可以视作是开源精神在企业内部的延伸，也是技术民主化的又一次跃迁。

因此，所谓“业务 + 科技”共建，也并不是一种全新的概念，而是传统开源协作理念在企业内部的进一步普及与扩展。

随着这一趋势的加速发展，IT 从业者正面临一个前所未有的挑战：一个原本由 IT 主导的网络，正在被一批“跨界者”打破。这些人才既懂业务，又具备一定的技术能力，借助智能编程、低代码、零代码等 AI 工具，能够独立完成大量原本需要专业 IT 人员才能完成的工作。

我们来看几个具体例子，跨界者能够通过 Dify、Coze、AutoGen 等现有平台，快速搭建流程类智能体，或者依托 DB-GPT 等工具，能够快速完成数据分析类任务。而未来，随着 Manus、DataAgent、DeepResearch 等 “目标类智能体” 的技术成熟，其能覆盖的工作场景还将进一步拓展，对传统 IT 工作的重塑也会更深入。

这种变化也将带来一个看似矛盾但又会真实存在的现象：IT 需求总量可能上升，但需要专业开发的传统系统却会减少。

原因在于，信息流转速度的持续提升，对系统的响应能力和灵活性提出了更高的要求。而满足这些需求的关键，在于不再依赖传统的“指令控制式开发”，而是转向分布式节点的协同开发。在这样的模式下，业务需求将更加碎片化、实时化。业务人员可以借助智能工具，快速响应变化，不再需要长期等待 IT 部门的介入与交付。

这也意味着，传统的“开发—交付”模式将被打破。取而代之的，是一种全新的系统构建方式，所有相关方处于一个更扁平、更敏捷、更协同的网络中，共同参与系统的设计与演化。

那么，接下来的问题是，只有“人”会受到这种“极致个体 + 极致协同”趋势的影响吗？

当然不是。在这个不断加速的信息网络中，每一个节点，无论是人还是系统，都无法置身事外。

现在，让我们回到专栏中我多次强调的几个架构设计思想，重新审视它们背后的逻辑与演进方向。

### 一、面向智能体的设计

未来，企业内部将涌现出各式各样的智能体，它们能够自主运行、协同交互。智能体本身代表着“极致的个体”——具备独立感知、决策与执行能力，是在传统微服务基础上的演化；而多个智能体之间的协作，则体现了“极致的协同”，通过交互网络的构建释放出更强的系统性能力。

### 二、记忆框架的“无边界”设计

记忆框架的“无边界”设计，其核心思想正是对“极致协同”的追求。它强调信息的流动、不受边界限制，节点之间可以自由地共享与交互，从而实现更高效的协同。

### 三、实体 Token 的新概念

最后，再来看实体 Token 的构建，在学习过程中，许多人对这个概念也存在一些疑问。它本身是一个“极致的个体”。但是，不同之处在于，它不仅承载了多维度的数据，里面还包含了丰富的业务语义。从某种意义上说，实体 Token 实际上自身就是一个“极致个体”与“极致协同”的融合体。

而且，“实体 Token”这种模式的发展，也能为上层智能体的繁荣提供充分的“养料”，因此我认为它是未来企业实现智能原生生态的一个非常好的抓手。

可以看到，这几种设计思想的背后，其实都指向了同一个构建逻辑。而这种构建逻辑，也正是我们课程中另一个主角——大模型的本质。

在大模型中，Token 是最基本的信息单元，它被设计为尽可能承载丰富的语义信息——这正是“极致的个体”在技术层面的体现。而多个 Token 之间的组合、关联与协同，又构成了语言理解、逻辑推理，甚至是智能涌现的基础——这正是“极致的协同”的体现。

可以说，“极致个体 + 极致协同”是一种普遍存在的演化规律，是智能、复杂性与进化背后的共性逻辑，在上文所说的现实世界和技术世界，乃至自然生态世界中都是如此。

现在，我们简单总结一下，大模型对 IT 的真正冲击，并不只是一次技术的颠覆，而是一次系统思维的重构。它也将逐步推动打破传统 IT 架构设计中的各种“小空间”、“墙”或“边界思维”，走向一个更加开放、分布、协同的演进方向。

当然，对 IT 从业者来说，最关键的改变也不是急于去掌握多少项新技术，而是思维方式的转变。理解了这一趋势，不仅有助于我们把握未来架构设计的核心思想，也将决定我们在 AI 时代中的发展方向与核心竞争力。

## 后续学习建议

最后，我再结合前面课程提出的核心架构设计思想，提出几点后续行动建议，供你参考。

[![](https://static001.geekbang.org/resource/image/e4/97/e4afd741860cf930a7c1fc034ea58997.jpg?wh=4000x2250)](https://static001.geekbang.org/resource/image/e4/97/e4afd741860cf930a7c1fc034ea58997.jpg?wh=4000x2250)

### 一、渐进式落地智能体，践行“面向智能体设计”理念

智能体是未来的长期赛道，但直接落地 DataAgent、DeepResearch 等目标级智能体难度较高。因为，其不仅依赖工具与平台，更需企业足够的 “内功” 积累。

更关键的是，若仅将智能体局限于工具或技术范畴，会错失 “面向智能体设计” 新范式带来的巨大红利，这也是我们课程里反复强调的。

建议你认真体会“面向智能体设计”的理念，结合企业现状，从立体感知、主动感知、记忆框架、决策等环节逐步优化。即便目标级智能体无法一步到位，也可以通过“微卡片 + 智能体”等中间形态进行过渡，慢慢向高阶智能体演进。

### 二、吃透“实体 Token”，将数据变成一个个“智能单元”

希望你深入理解 “实体 Token” 概念。实际上，它的本质是一种高阶  “智能体”，能独立完成 “感知 - 决策 - 行动” 闭环，所需数据与 “推理逻辑” 均内置于自身。这与大模型的逻辑高度类似，大模型内部同样包含数据与推理逻辑。

此外，落地时，建议先在局部业务场景中推广“实体 Token”。这样可以在不触碰全局数据架构的前提下，实现局部范围内横向无边界的数据流通，为业务和智能体应用逐步沉淀可复制的案例。

### 三、把“记忆无边界”做成对抗概率输出的护城河

前面提到的  “记忆无边界” 的思想需要重点关注。现在，这种无边界空间也已延展至代码空间、文档空间等领域，这是应对大模型“概率式输出” 的有效解决方案。

对企业而言，未来这类无边界空间覆盖范围越广，智能能够发挥的价值空间也会越大。

### 四、系统学习 AI 与大模型，将思想反哺到架构设计

大模型及 AI 相关技术需要系统学习，深入理解其背后原理与模式。这是因为，大模型不仅是一次技术浪潮，更是凝结了人类顶级智慧的系统性解决方案。而架构设计的本质也是用系统性思维解决现实问题。

因此，后续学习中，建议大家不仅要关注大模型带来的技术与工具革新，更要多追问“这项技术到底压缩了现实世界的哪一类复杂性”，并将其映射到自身架构的痛点与约束，做到“用 AI 思想赋能架构设计”。

愿你能将大模型蕴育的新理念变为你的设计直觉，让它在每一次权衡、每一张蓝图里自然流淌，完成向 AI 时代架构师的跃迁。
