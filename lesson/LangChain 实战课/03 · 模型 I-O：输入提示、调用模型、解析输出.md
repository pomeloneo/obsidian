## 提示模板

语言模型是个无穷无尽的宝藏，人类的知识和智慧好像都封装在了这个"魔盒"里面。但怎样才能解锁其中的奥秘，那可就是仁者见仁智者见智了。所以现在"**提示工程**"这个词特别流行，所谓 Prompt Engineering，就是专门研究对大语言模型的提示构建。

> [!important]
> 
> **吴恩达老师的提示工程原则**：
> 
> 1. 给予模型**清晰明确**的指示
> 
> 1. 让模型**慢慢地思考**

道理总是简单，但如何具体实践这些原则，又是个大问题。让我从创建一个简单的 LangChain 提示模板开始。

### 创建提示模板

我们希望为销售的每一种鲜花生成一段简介文案。每当员工或顾客想了解某种鲜花时，调用该模板就会生成适合的文字。

```python
from langchain.prompts import PromptTemplate

template = """您是一位专业的鲜花店文案撰写员。\n
对于售价为 {price} 元的 {flower_name} ，您能提供一个吸引人的简短描述吗？
"""

prompt = PromptTemplate.from_template(template)
print(prompt)
```

**输出结果**：

```jsx
input_variables=['flower_name', 'price']
output_parser=None 
partial_variables={}
template='\n您是一位专业的鲜花店文案撰写员。\n对于售价为 {price} 元的 {flower_name} ，您能提供一个吸引人的简短描述吗？\n'
template_format='f-string'
validate_template=True
```

> [!important]
> 
> **模板解析**：
> 
> - 模板是一个 **f-string**，包含两个占位符 `{flower_name}` 和 `{price}`
> 
> - `from_template` 是类方法，将原始字符串模板转化为 `PromptTemplate` 对象
> 
> - 对象中包含：输入变量、输出解析器、模板格式、是否验证模板等信息

## 语言模型

LangChain 中支持的模型有**三大类**：

### 调用模型生成文案

```python
import os
os.environ["OPENAI_API_KEY"] = '你的Open AI API Key'

from langchain_openai import OpenAI

model = OpenAI(model_name='gpt-3.5-turbo-instruct')
input = prompt.format(flower_name=["玫瑰"], price='50')
output = model.invoke(input)
print(output)
```

`prompt.format()` 将模板实例化，把 `{flower_name}` 替换为 "玫瑰"，`{price}` 替换为 '50'，形成具体提示：

> "您是一位专业的鲜花店文案撰写员。对于售价为 50 元的玫瑰，您能提供一个吸引人的简短描述吗？"

**模型输出**：

```jsx
让你心动！50元就可以拥有这支充满浪漫气息的玫瑰花束，让TA感受你的真心爱意。
```

### 批量生成多种鲜花文案

```python
from langchain import PromptTemplate

template = """您是一位专业的鲜花店文案撰写员。\n
对于售价为 {price} 元的 {flower_name} ，您能提供一个吸引人的简短描述吗？
"""
prompt = PromptTemplate.from_template(template)

import os
os.environ["OPENAI_API_KEY"] = '你的Open AI API Key'

from langchain import OpenAI
model = OpenAI(model_name='gpt-3.5-turbo-instruct')

flowers = ["玫瑰", "百合", "康乃馨"]
prices = ["50", "30", "20"]

for flower, price in zip(flowers, prices):
    input_prompt = prompt.format(flower_name=flower, price=price)
    output = model.invoke(input_prompt)
    print(output)
```

**输出**：

```jsx
这支玫瑰，深邃的红色，传递着浓浓的深情与浪漫，令人回味无穷！
百合：美丽的花朵，多彩的爱恋！30元让你拥有它！
康乃馨—20元，象征爱的祝福，送给你最真挚的祝福。
```

### LangChain vs 直接调用 OpenAI API

你可能会问：直接调用 OpenAI API 不是完全可以实现相同功能吗？

**直接使用 OpenAI API**：

```python
import openai
openai.api_key = 'Your-OpenAI-API-Key'

prompt_text = "您是一位专业的鲜花店文案撰写员。对于售价为{}元的{}，您能提供一个吸引人的简短描述吗？"

flowers = ["玫瑰", "百合", "康乃馨"]
prices = ["50", "30", "20"]

for flower, price in zip(flowers, prices):
    prompt = prompt_text.format(price, flower)
    response = openai.completions.create(
        engine="gpt-3.5-turbo-instruct",
        prompt=prompt,
        max_tokens=100
    )
    print(response.choices[0].text.strip())
```

> [!important]
> 
> **LangChain 的优势**：
> 
> - **代码可读性**：模板更易于阅读和理解
> 
> - **可复用性**：模板可在多处复用，代码更简洁
> 
> - **维护性**：修改提示只需修改模板
> 
> - **变量处理**：自动处理变量插入
> 
> - **参数化**：根据不同参数生成不同提示
> 
> - **模型切换**：无需修改代码即可切换模型

### 切换到 HuggingFace 模型

使用完全相同的提示模板，发送给 HuggingFaceHub 中的开源模型：

[![](https://static001.geekbang.org/resource/image/c8/ec/c8c0d84349ebd2d1d82a2836383164ec.png?wh=1704x854)](https://static001.geekbang.org/resource/image/c8/ec/c8c0d84349ebd2d1d82a2836383164ec.png?wh=1704x854)

HuggingFace Hub 界面

```python
from langchain.prompts import PromptTemplate

template = """You are a flower shop assitiant.\n
For {price} of {flower_name} ，can you write something for me？
"""
prompt = PromptTemplate.from_template(template)

import os
os.environ['HUGGINGFACEHUB_API_TOKEN'] = '你的HuggingFace API Token'

from langchain_community.llms import HuggingFaceHub
model = HuggingFaceHub(repo_id="google/flan-t5-large")

input = prompt.format(flower_name=["rose"], price='50')
output = model(input)
print(output)
```

**输出**：

```jsx
i love you
```

> [!important]
> 
> 一分钱一分货！使用较早期的开源模型 T5，得到了很粗糙的文案。另外 T5 还没有支持中文的能力，所以提示文字换成了英文。
> 
> **核心信息**：你可以重用模板、重用程序结构，通过 LangChain 框架调用**任何模型**。

## 输出解析

LangChain 提供的解析模型输出的功能，使你能够更容易地从模型输出中获取**结构化的信息**。

### 为什么需要输出解析？

前面的例子中，模型只生成了一段文案字符串。但在实际开发中，我们更多需要的是程序能够直接处理的**结构化数据**。

比如希望模型返回两个字段：

- **description**：鲜花的说明文本

- **reason**：解释为何这样写文案

从笼统言语到结构清晰的数据，这就需要 **LangChain 的输出解析器**。

### 完整代码示例

```python
import os
os.environ["OPENAI_API_KEY"] = '你的OpenAI API Key'

from langchain.prompts import PromptTemplate

prompt_template = """您是一位专业的鲜花店文案撰写员。
对于售价为 {price} 元的 {flower_name} ，您能提供一个吸引人的简短描述吗？
{format_instructions}"""

from langchain_openai import OpenAI
model = OpenAI(model_name='gpt-3.5-turbo-instruct')

# 定义输出结构
from langchain.output_parsers import StructuredOutputParser, ResponseSchema

response_schemas = [
    ResponseSchema(name="description", description="鲜花的描述文案"),
    ResponseSchema(name="reason", description="问什么要这样写这个文案")
]

# 创建输出解析器
output_parser = StructuredOutputParser.from_response_schemas(response_schemas)
format_instructions = output_parser.get_format_instructions()

# 创建整合了输出解析结构的提示模板
prompt = PromptTemplate.from_template(
    prompt_template,
    partial_variables={"format_instructions": format_instructions}
)

# 批量生成并解析
flowers = ["玫瑰", "百合", "康乃馨"]
prices = ["50", "30", "20"]

import pandas as pd
df = pd.DataFrame(columns=["flower", "price", "description", "reason"])

for flower, price in zip(flowers, prices):
    input = prompt.format(flower_name=flower, price=price)
    output = model.invoke(input)
    parsed_output = output_parser.parse(output)
    parsed_output['flower'] = flower
    parsed_output['price'] = price
    df.loc[len(df)] = parsed_output

print(df.to_dict(orient='records'))
df.to_csv("flowers_with_descriptions.csv", index=False)
```

**输出**：

```python
[
    {
        'flower': '玫瑰', 
        'price': '50', 
        'description': 'Luxuriate in the beauty of this 50 yuan rose...', 
        'reason': 'This description emphasizes the elegance and beauty...'
    },
    {
        'flower': '百合', 
        'price': '30', 
        'description': '30元的百合，象征着坚定的爱情，带给你的是温暖而持久的情感！', 
        'reason': '百合是象征爱情的花，写出这样的描述能让顾客更容易感受到百合所带来的爱意。'
    },
    {
        'flower': '康乃馨', 
        'price': '20', 
        'description': 'This beautiful carnation is the perfect way to show your love...', 
        'reason': 'The description is short, clear and appealing...'
    }
]
```

> [!important]
> 
> **代码解析**：
> 
> 1. 定义 `response_schemas` 列表，包含 description 和 reason 两个字段
> 
> 1. 通过 `StructuredOutputParser.from_response_schemas` 创建输出解析器
> 
> 1. 用 `get_format_instructions()` 获取格式说明，整合到提示模板
> 
> 1. 用 `output_parser.parse(output)` 把模型输出解析成 Python 字典

### 解析后的数据结构

## 总结时刻

恭喜你从头到尾利用大模型开发出了一个**能够自动生成鲜花文案的应用程序**！

> 每一次运行都有不同的结果，我们完全不知道大模型下一次会给我们带来怎样的新东西。基于大模型构建的应用可以说**充满了创造力**。

> [!important]
> 
> **LangChain 框架优势总结**：
> 
> - **模板管理**：帮助管理大型项目中的多个提示模板
> 
> - **变量提取和检查**：自动提取并检查模板中的变量
> 
> - **模型切换**：更改模型名称即可切换，无需修改代码
> 
> - **输出解析**：提示模板可嵌入输出格式定义，方便后续处理

下节课我们将继续深入探索 LangChain 中的提示模板，看如何通过高质量的提示工程让模型创造出更精准、更高质量的输出。
