04｜如何在前端调用语音合成和视觉模型

你好，我是月影。

我们知道，AI 大模型应用的输入、输出通常是多模态的，不论大模型本身是否支持不同格式的输入输出，在业务上，我们都能够通过调用不同的服务整合来做到这一点。

在这节课里，我们就来体验一下**火山引擎**和**月之暗面**的服务，通过具体实践了解如何使用语音合成和视觉模型。

### 创建项目

在 Trae 中创建一个 Vue 项目并添加 `.env.local`：

```jsx
VITE_APP_ID=5934290469
VITE_ACCESS_TOKEN=c-*********Ln4N
VITE_CLUSTER_ID=volcano_tts
```

### 实现 App.vue

```jsx
<script setup lang="ts">
import { ref } from 'vue';

const prompt = ref('您好，请问有什么可以帮您？');
const status = ref('ready');
const audioEl = ref<HTMLAudioElement>();

function createBlobURL(base64AudioData: string): string {
  var byteArrays = [];
  var byteCharacters = atob(base64AudioData);
  for (var offset = 0; offset < byteCharacters.length; offset++) {
    var byteArray = byteCharacters.charCodeAt(offset);
    byteArrays.push(byteArray);
  }
  var blob = new Blob([new Uint8Array(byteArrays)], { type: 'audio/mp3' });
  return URL.createObjectURL(blob);
}

const generateAudio = async () => {
  const token = import.meta.env.VITE_ACCESS_TOKEN;
  const appId = import.meta.env.VITE_APP_ID;
  const clusterId = import.meta.env.VITE_CLUSTER_ID;
  const voiceName = "zh_female_shuangkuaisisi_moon_bigtts";

  const endpoint = '/tts/api/v1/tts';
  const headers = {
    'Content-Type': 'application/json',
    Authorization: `Bearer;${token}`,
  };

  const payload = {
    app: {
      appid: appId,
      token,
      cluster: clusterId,
    },
    user: {
      uid: 'bearbobo',
    },
    audio: {
      voice_type: voiceName,
      encoding: 'ogg_opus',
      compression_rate: 1,
      rate: 24000,
      speed_ratio: 1.0,
      volume_ratio: 1.0,
      pitch_ratio: 1.0,
      emotion: 'happy',
    },
    request: {
      reqid: Math.random().toString(36).substring(7),
      text: prompt.value,
      text_type: 'plain',
      operation: 'query',
      silence_duration: '125',
      with_frontend: '1',
      frontend_type: 'unitTson',
      pure_english_opt: '1',
    },
  };

  status.value = 'generating';
  const res = await fetch(endpoint, {
    method: 'POST',
    headers,
    body: JSON.stringify(payload),
  });

  const data = await res.json();
  if (!data.data) {
    throw new Error(JSON.stringify(data));
  }

  const url = createBlobURL(data.data);
  audioEl.value && (audioEl.value.src = url)
  audioEl.value?.play();
  status.value = 'done';
};
</script>

<template>
  <div class="container">
    <div>
      <label>Prompt </label>
      <button @click="generateAudio">Generate & Play</button>
    </div>

      <audio ref="audioEl"></audio>
    </div>
  </div>
</template>
```

### 代码解析

**1. 读取配置与设置音色**

```jsx
const token = import.meta.env.VITE_ACCESS_TOKEN;
const appId = import.meta.env.VITE_APP_ID;
const clusterId = import.meta.env.VITE_CLUSTER_ID;
const voiceName = "zh_female_shuangkuaisisi_moon_bigtts";
```

**2. 配置 Vite 代理**

直接调用火山引擎服务会有跨域问题，需要修改 `vite.config.ts` 添加 server 代理：

```tsx
server: {
  allowedHosts: true,
  proxy: {
    '/tts': {
      target: 'https://openspeech.bytedance.com',
      changeOrigin: true,
      rewrite: path => path.replace(/^\/tts/, ''),
    }
  },
},
```

**3. 请求头与 Payload**

```jsx
const endpoint = '/tts/api/v1/tts';
const headers = {
  'Content-Type': 'application/json',
  Authorization: `Bearer;${token}`,
};

const payload = {
  app: { appid: appId, token, cluster: clusterId },
  user: { uid: 'bearbobo' },
  audio: {
    voice_type: voiceName,
    encoding: 'ogg_opus',
    compression_rate: 1,
    rate: 24000,
    speed_ratio: 1.0,
    volume_ratio: 1.0,
    pitch_ratio: 1.0,
    emotion: 'happy',
  },
  request: {
    reqid: Math.random().toString(36).substring(7),
    text: prompt.value,
    text_type: 'plain',
    operation: 'query',
    silence_duration: '125',
    with_frontend: '1',
    frontend_type: 'unitTson',
    pure_english_opt: '1',
  },
};
```

**语音参数说明：**

|参数|说明|
|---|---|
|`voice_type`|音色，火山引擎支持数十种音色，示例用"爽快思思"。包含非中文内容时应选择多语言音色|
|`encoding`|语音格式（mp3、ogg 等），需考虑终端兼容性|
|`compression_rate`|音频压缩率，影响文件大小和质量|
|`rate`|采样率（Hz），24000 Hz 是常用值，音质较好|
|`speed_ratio`|语速，1.0 为正常语速|
|`volume_ratio`|音量，1.0 为正常音量|
|`pitch_ratio`|音调，1.0 为正常音调|
|`emotion`|情感表达，`happy` 表示欢快语气|

**4. 发起请求与播放音频**

```jsx
status.value = 'generating';
const res = await fetch(endpoint, {
  method: 'POST',
  headers,
  body: JSON.stringify(payload),
});

const data = await res.json();
if (!data.data) {
  throw new Error(JSON.stringify(data));
}

const url = createBlobURL(data.data);
audioEl.value && (audioEl.value.src = url)
audioEl.value?.play();
status.value = 'done';
```

返回数据中 `data` 字段是音频的 Base64 编码，通过 `createBlobURL` 函数转换为二进制 Blob 对象，生成 URL 并设置为 `<audio>` 标签的 `src`，然后执行 `play()` 播放。

```jsx
function createBlobURL(base64AudioData: string): string {
  var byteArrays = [];
  var byteCharacters = atob(base64AudioData);
  for (var offset = 0; offset < byteCharacters.length; offset++) {
    var byteArray = byteCharacters.charCodeAt(offset);
    byteArrays.push(byteArray);
  }
  var blob = new Blob([new Uint8Array(byteArrays)], { type: 'audio/mp3' });
  return URL.createObjectURL(blob);
}
```

> 在实际产品中，通常不会直接将任意文字转语音，而是将文本模型生成的回答内容转为语音，组成特定的智能体来实现应用。关于这部分内容，后续课程会进一步深入探讨。

## 使用 Kimi 视觉模型

在有些 AI 应用中，我们不仅仅让用户提供文字信息，还可以让用户提供图像信息。这时候，**视觉（Vision）模型**就起到了分析和理解图片内容的重要作用。

这里我们以月之暗面的 **Kimi** 为例，来学习如何使用视觉模型。

### 注册与获取 API Key

在 [platform.moonshot.cn](http://platform.moonshot.cn) 完成注册，进入控制台。Kimi 和之前了解的 DeepSeek 平台差不多，点击左侧菜单选择 **API Key 管理**，新建一个 API Key。

> [!important]
>
> Kimi 的视觉模型包括：
>
> - `moonshot-v1-8k-vision-preview`
>
> - `moonshot-v1-32k-vision-preview`
>
> - `moonshot-v1-128k-vision-preview`
>
> 它们能力基本一样，区别是 token 的数量限制。本例使用 `moonshot-v1-8k-vision-preview`。

### 创建项目

使用 Trae 创建项目 `Kimi Vision Demo`，添加 `.env.local`：

```jsx
VITE_API_KEY=sk-qi2o**********xbp4
```

### 实现 App.vue

```jsx
<script setup lang="ts">
import { ref, computed } from 'vue';

const content = ref('');
const imgBase64Data = ref('');
const isValid = computed(() => imgBase64Data.value !== '');

const updateBase64Data = async (e: Event) => {
  imgBase64Data.value = '';
  const file = (e.target as HTMLInputElement).files?.[0];
  if (!file) {
    return;
  }
  const reader = new FileReader();
  reader.readAsDataURL(file);
  reader.onload = () => {
    imgBase64Data.value = reader.result as string;
  };
};

const update = async () => {
  if (!imgBase64Data.value) {
    return;
  }
  const endpoint = 'https://api.moonshot.cn/v1/chat/completions';
  const headers = {
    'Content-Type': 'application/json',
    Authorization: `Bearer ${import.meta.env.VITE_API_KEY}`
  };

  content.value = '思考中…'
  const response = await fetch(endpoint, {
    method: 'POST',
    headers: headers,
    body: JSON.stringify({
      model: 'moonshot-v1-8k-vision-preview',
      messages: [
        { role: 'user',
          content: [{
            type: "image_url",
            image_url: {
              "url": imgBase64Data.value,
            },
          }, {
            type: "text",
            text: "请描述图片的内容。",
          }]
        }
      ],
      stream: false,
    })
  });

  const data = await response.json();
  content.value = data.choices[0].message.content;
};
</script>

<template>
  <div class="container">
    <div>
      <label>文件：</label>
      <input class="input" type="file"
        accept=".jpg, .jpeg, .png, .gif"
        @change="updateBase64Data"/>
      <button @click="update" :disabled="!isValid">提交</button>
    </div>
    <div class="output">
      <div class="preview">
        <img v-if="imgBase64Data" :src="imgBase64Data" alt="preview" />
      </div>
      <div> content </div>
    </div>
  </div>
</template>

<style scoped>
.container {
  display: flex;
  flex-direction: column;
  align-items: start;
  justify-content: start;
  height: 100vh;
  font-size: .85rem;
}
.input {
  width: 200px;
}
.output {
  margin-top: 10px;
  min-height: 300px;
  width: 100%;
  text-align: left;
}
.preview img {
  max-width: 100%;
}
button {
  padding: 0 10px;
  margin-left: 6px;
}
</style>
```

### 代码解析

和调用文本模型的区别很小，只是输入由文本内容换成了图片内容。

**1. 获取图片 Base64 数据**

使用 `FileReader` 在浏览器端直接获取图片的 Base64 数据：

```jsx
const updateBase64Data = async (e: Event) => {
  imgBase64Data.value = '';
  const file = (e.target as HTMLInputElement).files?.[0];
  if (!file) return;

  const reader = new FileReader();
  reader.readAsDataURL(file);
  reader.onload = () => {
    imgBase64Data.value = reader.result as string;
  };
};
```

`reader.readAsDataURL` 会将文件内容解析为 Base64 编码字符串，并带上格式头（如 `data:image/png;base64,...`），可以直接传给 Kimi 的多模态大模型。

**2. 发送多模态请求**

Kimi 多模态模型支持 `type` 为 `image_url` 的内容，`image_url` 字段可以接收图片的 Base64 数据：

```jsx
const response = await fetch(endpoint, {
  method: 'POST',
  headers: headers,
  body: JSON.stringify({
    model: 'moonshot-v1-8k-vision-preview',
    messages: [
      { role: 'user',
        content: [{
          type: "image_url",
          image_url: { "url": imgBase64Data.value },
        }, {
          type: "text",
          text: "请描述图片的内容。",
        }]
      }
    ],
    stream: false,
  })
});
```

这样就实现了视觉识别，让应用具有了接受和分析图片内容的能力。

## 要点总结

> [!important]
>
> - **语音合成**：将文本通过大模型转换为带有类似真人情感的语音，在前端播放
>
> - **视觉模型**：接受图片输入，按照用户要求分析图片内容并以文字描述
>
> - 这两种能力的引入，对于实现**多模态 AI 应用**非常有帮助

## 课后练习

### 练习一：探索不同音色

火山引擎的语音模型支持数十种不同的音色。修改例子给界面增加音色选择功能，对同样的文本合成不同音色的语音效果，同时配合修改 `emotion` 情感参数，体验它们的区别。

### 练习二：思考 Base64 转 Blob 的方式

在语音合成实践中，我们采用了将 Base64 字符串转换为 Blob 的方式来播放语音。为什么这么做？有什么好处？有没有其他可行的办法？

### 练习三：扩展视觉模型能力

在视觉模型的例子里，我们固定要求 AI 描述图片内容。实际上还可以通过指令控制 AI 做其他处理，比如对物品进行分类、说出主要物品的英文单词、看图说话撰写作文等。请修改例子，增加用户输入框，让用户可以输入不同的要求。
