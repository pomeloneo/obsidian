07｜提示词工程（一）：三个技巧让大模型超常发挥

你好，我是月影。

要让大模型发挥最大的效果，**提示词（Prompt）**的设计是很关键的一件事，优秀的提示词不仅能让大模型输出更好的结果，还能节省 token，降低成本。

前面的课程中，我们已经通过实战体会过一些提示词的使用。在这一节课里，我将为你系统地梳理一下书写提示词的常用技巧，帮助你更好地掌控大模型的能力。

---

## 如何正确书写文本提示词

对于大部分文本大模型而言，实际上是具备理解复杂提示词的能力的，尤其是大参数的模型。在这个情况下，书写有效提示词最重要的一点非常简单——**说人话**。

也就是说，你可以把 AI 当作"人类"，给它明确清晰的指令就可以了。

比如用 DeepSeek-R1 模型时，你可以直接提出原始需求：

DeepSeek-R1 直接输入需求

DeepSeek-R1 有能力进行深度思考，所以它会自己分析推理，并给出思考过程：

- DeepSeek-R1 的思考过程（展开查看）

    好的，我现在需要帮用户规划一个前端入门课程的大纲。用户是给初学者讲课，所以需要从最基础的内容开始，逐步深入，同时要确保课程结构合理，内容全面。首先，我得考虑初学者的背景，他们可能对编程了解不多，所以得从最基础的开始，比如 HTML、CSS，然后逐步引入 JavaScript，再到框架和工具。

    首先，前端开发的基础肯定是 HTML 和 CSS，这两者是构建网页的基石。应该先讲 HTML 的基本结构、常用标签，然后是 CSS 的选择器、盒模型、布局技术。这部分可能需要几周的时间，因为初学者需要时间来练习和熟悉这些概念。接下来是 JavaScript，这是让网页动起来的关键。需要从变量、数据类型、函数等基础讲起，然后过渡到 DOM 操作和事件处理，这样学生可以实际看到自己的代码在页面上产生的效果。

    然后，响应式设计和 CSS 框架也是必须的，比如媒体查询和 Flexbox、Grid 布局，以及 Bootstrap 或 Tailwind 这样的框架，这些内容能帮助学生快速构建美观的网页。接下来，JavaScript 的高级部分，比如 ES6+ 特性、异步编程、模块化，还有 AJAX 和 Fetch API，这些是开发动态网页和与后端交互的基础。

    之后，版本控制工具 Git 和 GitHub 的使用也是必不可少的，因为在实际开发中协作和代码管理非常重要。然后，前端构建工具如 Webpack、Babel，以及包管理器 npm/yarn，这些工具能让学生了解现代前端工作流程。接下来是前端框架，Vue.js、React 或 Angular，选择一个作为入门，这里可能需要更多时间，因为框架的学习曲线较陡，尤其是组件化开发和状态管理。

    项目实战部分非常重要，通过实际项目让学生综合运用所学知识，比如 TodoList、博客系统或电商网站首页，这能帮助他们巩固知识并积累经验。最后，部署和优化，以及进阶方向，帮助学生了解如何将项目上线，并指引他们未来的学习路径。

    - 开发一个完整响应式网站（个人作品集/电商展示页/新闻资讯站）

通过上面的实践，我们会看到，对于功能强大的模型（如 DeepSeek-R1），你只要给它**准确清晰的指令**，它通过深入思考，通常就会做得非常优秀。

> [!important]
>
> 注意你仍然需要给大模型尽量**清晰准确的指令**。例如"初学者、入门课"就是非常重要的信息，它告诉了大模型期待的课程难度和适用人群。

不过，DeepSeek-R1 的深度思考模式需要花费更多的算力和推理时间，价格也更贵。如果想在**能力弱一些、参数规模较小的模型**下依然得到不错的结果，就需要在提示词上多花功夫了。

比如换成豆包，输出结果虽然也还不错，但相对来说不如 DeepSeek-R1。如果要得到更好的结果，以下三种技巧可以提升效果。

## 技巧一：使用结构化提示词

**提示词框架（Prompt Framework）**是指设计一组清晰且结构化的提示词，用以引导模型生成特定类型的输出。它有助于提高生成的准确性、相关性和质量。

提示词框架有很多种，有些适用于特定场景（如后续课程的"AI 面试官"会用 **RCSA** 框架）。这里我们用一个简单的结构化提示词来改写输入。

使用豆包模型，在 Coze 上创建智能体，在"人设与回复逻辑"输入：

> 你是一位曾经就职于互联网头部企业的资深软件工程师和IT教育专家，擅长用通俗易懂的语言来给初学者讲解相应技术方向的基本概念和入门知识。

> },

> 根据用户的输入，整理一门入门级技术课程的大纲，要求：

> 1. 注重基本概念和原理，为学员打下扎实基础

> 2. 具有实操性，注重实用性

> 3. 内容不要求大而全，要循序渐进，适合初级学员掌握

> 4. 准备的案例简单而兼具趣味与挑战

> 5. 展望未来，适当介绍一些新技术和技术趋势

这是一个典型的结构化提示词，采用了**角色（Role）→ 任务（Task）→ 结果（Result）**的基础结构。通过这样的设定，约束了大模型的输出，从而得到更好的结果。

> [!important]
>
> 有兴趣的同学可以用上面的结构化提示词，分别调用 DeepSeek、豆包和 Kimi，再对比不用结构化提示词的情况，看一下它们的差异。

## 技巧二：使用分步思考工作流

在不使用深度思考模型的前提下，我们可以利用**工作流**让 AI 进行深度思考。

还是前端课程大纲的例子，创建一个 Coze 工作流，添加一个大模型节点，负责思考课程设计思路。

[![](https://static001.geekbang.org/resource/image/2c/ab/2c4e046ffbc0b869ba00ca25bdc0a6ab.png?wh=1804x1200)](https://static001.geekbang.org/resource/image/2c/ab/2c4e046ffbc0b869ba00ca25bdc0a6ab.png?wh=1804x1200)

分步思考工作流

**节点一：课程设计思路（系统提示词）**

> 你是一位曾经就职于互联网头部企业的资深软件工程师，拥有丰富的软件开发经验。

> 你根据用户需求，为授课老师设计满足该需求的入门课程提供经验指导。

> “subtopic”: “光的组成”,

> 提供经验指导的具体方法如下：

> - 以你过往的工作经验，分析该课程的核心重点和教学难点

> - 考虑学员是零基础的初学者，需要由浅入深的上手指南

> - 指导授课老师如何让课程内容有实用性，不与实际工作脱节

> - 协助授课老师在课程中适当增加实战经验

> - 让课程有实操性，案例有一定挑战但不会让学员觉得太难

> - 协助授课老师考虑课堂与学员的随堂互动

> “explaination_in_english”: “The Earth’s atmosphere is filled with various gas molecules, such as nitrogen and oxygen, as well as some tiny dust particles. When sunlight encounters these gas molecules and dust, scattering occurs. Among them, Rayleigh scattering is an important scattering method, and its scattering intensity is inversely proportional to the fourth - power of the wavelength of light. That is to say, light with a shorter wavelength is more likely to be scattered.”

> 你不需要输出完整课程或实际的课程大纲，只需要输出你的思路、要点和建议。

**节点二：输出课程大纲**

配置输入参数（`req` 和 `guide`），撰写提示词：

输入参数配置

> **系统提示词：** 你是一位IT教育专家，擅长用通俗易懂的语言来给初学者讲解基本概念和入门知识。根据用户需求和设计思路，整理并输出入门级技术课程的大纲。

连接节点完成工作流，点击试运行：

工作流试运行

> [!important]
>
> **分步思考工作流的优势：** 将任务颗粒度细化，让不同的智能体每次只专注于一件事，然后通过协作汇总完成，通常能较大程度**提升输出质量**。

> 由于大模型的注意力机制，一次交互更适合集中精力处理一件明确的事，单点处理结果比同时处理多件事效果更好。但一次处理多件事也有优点——能节省 token 数量和推理时间。我们可以使用下面的 JSON 输出技巧来实现。

## 技巧三：使用 JSON 输出

文本大模型有较强的推理能力，有时候我们不用描述太复杂的提示词，可以通过**语义化的 JSON 格式**来取巧地实现想要的效果。

### 例一：故事生成

创建 Coze 智能体，在"人设与回复逻辑"输入：

```jsx
根据toolu_01DjuhS3daxsKM2Y18JLZPd4的主题，用**中文**输出以下JSON格式内容：
{
  "story_instruction": "",
  "the_whole_story_content": "",
  "the_whole_story_translate_to_en": "",
  "lessons": []
}
```

输入"狼来了"，输出结果：

- JSON 输出结果（展开查看）

    ```json
    {
      "story_instruction": "讲述一个放羊娃多次撒谎喊'狼来了'，最后狼真的来了却没人相信他的故事，教育人们要诚实，不能说谎。",
      "the_whole_story_content": "从前，有个放羊娃，每天都去山上放羊。一天，他觉得十分无聊，就想了个捉弄大家寻开心的主意……",
      "the_whole_story_translate_to_en": "Once upon a time, there was a shepherd boy who went up the mountain to herd sheep every day...",
      "lessons": [
        "做人要诚实，不能通过说谎来达到自己的目的。",
        "如果经常说谎，当真正有危险需要帮助时，可能就不会有人相信和伸出援手。"
      ]
    }
    ```

只需要用**符合语义的 JSON 格式**作为输出格式，不需要额外解释，大模型就能输出符合预期的 JSON 结果。

### 例二：科学问答

更复杂一点的例子——根据用户询问的科学问题，输出科普式解答：

```jsx
根据用户toolu_01DjuhS3daxsKM2Y18JLZPd4要求，解释科学内容，用**中文内容**以JSON格式回复。
输出：
{
  "topic": "",
  "sections": [
    { "section": 1, "subtopic": "", "explaination": "", "explaination_in_english": "" },
    { "section": 2, "subtopic": "", "explaination": "", "explaination_in_english": "" },
    { "section": 3, "subtopic": "", "explaination": "", "explaination_in_english": "" },
    { "section": 4, "subtopic": "总结", "explaination": "", "explaination_in_english": "" }
  ],
  "next_question": ""
}
```

输入"天空为什么是蓝色的"，大模型输出了包含"光的组成"、"瑞利散射"、"蓝色光的散射"和"总结"四个章节的完整科普内容，并附带英文翻译和下一个推荐问题。

### 回顾上一节课

还记得上一节课"拍照记单词"中使用的提示词吗？也是运用 JSON 输出来获得结构化内容：

```json
{
  "image_discription": "图片描述",
  "representative_word": "图片代表的英文单词",
  "example_sentence": "结合英文单词和图片描述，给出一个简单的例句",
  "explaination": "结合图片解释英文单词...",
  "explaination_replys": ["回复1", "回复2"]
}
```

> [!important]
>
> 由于大模型对 JSON 结构化信息的分析和处理能力很强，采用 JSON 格式输出通常能获得比文本输出**更加优质的内容**。但 JSON 的结构特点使得在输出 JSON 的同时充分利用流式输出减少响应时间变得比较困难，这正是后续几节课要讨论并解决的核心问题。

## 要点总结

> [!important]
>
> - **说人话**：给大模型明确清晰的指令，尤其是大参数模型
>
> - **结构化提示词**：采用角色→任务→结果的框架，约束输出质量
>
> - **分步思考工作流**：将任务拆分为多个节点，每个节点专注一件事，提升输出质量
>
> - **JSON 输出**：通过语义化的 JSON 格式，让大模型输出结构化且高质量的内容
>
> - 提示词设计虽然不是传统编程技术，但对逻辑思维能力有要求，可以用技术思路去优化

### 练习一：改进分步思考工作流

尝试修改分步思考工作流，思考如何进一步提高内容输出质量。如果要依次输出大纲中每个章节的提纲、重点和章节中的具体目录，用 Coze 工作流怎么实现？

### 练习二：统一输出格式

运行多次工作流时，可能发现每次生成的大纲格式不相同，这不利于应用界面处理。有什么办法可以让它每次生成的大纲格式相同吗？
