## 为什么需要 AI Agent？

我们每天都在用各种 AI Agent，比如用 Cursor 写代码，用 Manus 做各种自动化的事情。

直接调大模型存在这些问题：

- **记忆有限** — 上周聊过的消息记不住

- **无法执行操作** — 只能告诉你思路，无法帮你访问网页、执行任务

- **无法访问私有数据** — 不知道你公司内部的私密文档

- **知识滞后** — 训练数据有截止时间，不知道最新发生的事

这些问题的解决需要分别给大模型加上：

|能力|作用|
|---|---|
|**Memory**|记忆管理，让模型记住关键信息|
|**Tool**|工具调用，让模型执行具体操作|
|**RAG**|知识库检索，让模型访问私有文档|

---

## Agent 是什么？

> Agent = 大模型 + Tool + Memory + RAG

给大模型扩展了 Tool 和 Memory 后：

- 它本来就可以 **思考、规划**

- 用 Tool 扩展能力 → 可以 **自动做事情**

- 用 Memory 管理记忆 → 可以 **记住关键信息**

- 用 RAG 查询知识库 → 可以 **获取内部知识**

这样一个知道内部知识、能思考规划、有记忆、能帮你做事的大模型，就是一个 **Agent**。

---

## 实际应用案例

### Cursor 等 AI IDE

- 读写文件、执行命令 → 扩展了 **Tool**

### Manus 自动化工具

- 打开浏览器、访问网页、点击元素、写入文件 → 扩展了 **Tool**

### 支付宝蚂小财

- 分析基金、推荐基金、获取基金数据 → 扩展了 **RAG**

## 技术栈选择

### 框架

|框架|用途|
|---|---|
|**LangChain**|开发单个 Agent，封装了 Tool/Memory/RAG 等能力|
|**LangGraph**|基于 LangChain，用于多 Agent 协作交互|

### 语言

- 框架有 Python 和 Node 版，概念和逻辑一致

- 前端可用 Node 版更顺滑上手

### 全栈技术

AI 代码在后端跑，还需要：

- **Redis** 存储记忆

- **MySQL** 存储知识

- **Nest** 作为 Node 后端框架

---

## 总结

> [!important]
> 
> **核心概念：** 给大模型扩展 Tool、Memory、RAG 等能力，让它可以执行具体任务，这就是 Agent。

**学习路径：**

1. LangChain — 开发单 Agent

1. LangGraph — 开发 Multi Agent

2. 结合后端技术 — 开发 AI 全栈产品
