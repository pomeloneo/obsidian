# 快速排序

\#数学&算法

其实快速排序和归并排序核心思想大体相似 也是不停地二分, 排序, 合并的过程

但是对于快速排序来说, **最重要的是快速排序会找一个值作为基准值**, 比这个值小的放在一起, 比这个值大的放在一起 这样每一次排序都可以确定这个基准值的位置是正确的

## 数学思维快排

数学思维写快排其实很简单 1. 选择某个元素作为基准元素, 然后小的元素放在基准元素的左边, 大的元素放在右边 2. 递归此操作, 知道给各个部分只剩下一个元素(递归终止条件)

实例代码

```
const quickSort = (arr) => {    const [pivot, …rest] = arr;    if (arr.length <= 1) return arr;    return [        …quickSort(rest.filter((i) => {return i <= pivot})),        pivot,        …quickSort(rest.filter((i) => {return i > pivot}))    ]}const arr1 = [5, 4, 3, 2, 1];const arr2 = [2, 1];const arr3 = [0, 1];const arr4 = [1, 1, 1, 1, 1];const arr5 = [1, 2, 3, 4, 5];console.log(quickSort(arr1))console.log(quickSort(arr2))console.log(quickSort(arr3))console.log(quickSort(arr4))console.log(quickSort(arr5))
```

## 人类思维快排

数学思维的快排缺点就是做了很多重复的操作 比如遍历操作 复制操作 人类思维写快排就是要减少遍历, 减少复制 使用就地操作

人类思维快排要满足这样几个需求 * 每次pivot元素要在数组正确的位置 * pivot前面的元素必须都要小于pivot但是不要求有序, pivot后面的同理 * 使用就地操作

要写一个辅助函数来实现上述需求

```
const handlerPivot = (arr, start, end) => {    if (end - start === 0) {return -1};    if (end - start === 1) {return 0};    const pivot = arr[start];    let smallEnd = start;   //  要从pivot位置开始, 因为一开始small数组没有元素    let bigStart = end; //  end位置是达不到的 因为是数组的length    let i = start + 1;  // 遍历的标记索引index    while (bigStart - smallEnd > 1) {        if (arr[i] < pivot) {            i++;            smallEnd++        } else {            bigStart -= 1;  // 由于end是达不到的所以swap之前要 -1            swap(arr, i, bigStart)        }    }    swap(arr, start, smallEnd);    return smallEnd;}const arr = [40, 100, 50, 90, 70, 0, 0, 1];const swap = (arr, a, b) => {    [arr[a], arr[b]] = [arr[b], arr[a]];}console.log(arr, handlerPivot(arr, 0, arr.length), 'end')
```

## 随机化快排

由于目前的代码每次都是选择第一位作为pivot元素

如果数组正好是逆序的 就会导致快排的时间复杂度退化到O(n^2), 平均是NlogN

所以就要引入一个随机选择pivot的机制来尽量避免这种情况

## 特点

特点: 分而治之, 降低问题规模(每次减少约等于一半)
