|补充医疗账单核算|只有一个出账规则：当月出当月的  <br>计费逻辑：按商户和人数*合同系统的月【销售单价】=月补充医疗账单||

### **2、账单-管理服务费账单**

|   |   |   |
|---|---|---|
|**管理服务费账单**|||
|类目|系统规则||
|管理服务费账单|账单日：每月1号出上月的费用，凌晨2点出，6点发布。（日期改不了，需要改，就只能线下手动导出）  <br>出具方式分为线上，线下出具，默认为线上出具。可在商户系统进行配置-流程品控。  <br>（线上变线下，无预警，只有单子流转，风控兜底）  <br>管理服务费账单计费规则分为  <br>一、 按“在职天数＞基准天数"人数  <br>计费逻辑  <br>在职人数*[39.9*（1+费率6.77%）]  <br>在职人数：在用工系统-对应的商户中的员工信息中取“在职中”“待离职”  <br>39.9：在合同系统-产品名称-服务费进行取值（不一定是39.9）  <br>费率6.77%：在合同系统-产品名称-代扣代缴费率  <br>   <br>二、 按"社保U发薪‘服务人数  <br>计费逻辑  <br>账期月以社保计费人员及发薪计费人员的“证件号码”取并集的人员，为计费人员  <br>①社保人数：上月“社保账单”中保险月份>=账期月份的汇缴、补缴人员  <br>②发薪人数：上月发放完成的自然人数；同人员同月多次发薪只收取一次  <br>   <br>账单出账失败的原因：①合同失效（账期时间与合同系统的【合同时间】不符合）  <br>②无主产品无法计算（合作了新产品但诶呦重新创建、合同系统配置的不是我们业务线，如招聘版的培训）③不支持比例收取的产品（如按流水进行收取配置的）  <br>出账失败的，会在用工系统的账单管理-失败梳理中显示，由账单组进行盘点  <br>   <br>线下出具：由客成组做账单，转账单组进行人工上传||
|管理服务费配置|商户配置  <br>①集团发薪，看集团关系 在哪儿看-————看账单管理配置关系  <br>只要发薪单有这个人 就会发薪，开启这个是 社保发薪取并的时候 将其他的主体重复的人员进行取消  <br>②同账期收费笔数：默认一笔，但账单组需要多笔。因为可能出现之前漏收了或者是出现了平移了等 需要再次发账单  <br>③线下导入商户与员工关系限制，线下上传管理服务费账单，限制一一对应（人不属于这个商户），不限制的场景是 需要多商户  <br>④尾差：有尾差（虚拟人模式磨平尾差），无尾差（非虚拟人模式）  <br>默认无尾差，无尾差的-用明细汇总的钱就是无尾差；有尾差-结算单代扣代缴就是有尾差||

### **1、社保账单更新逻辑**

|   |   |   |
|---|---|---|
|类目|系统规则||
|1、社保 账单日，约定 每月 1 号生成账单，推送到 101 和财务系统，这其中哪一步是需要人工操作|账单发布时间真实的时间是账单日的前一个账单日 ，当账单生成后，会先进入分毫系统，由人工按账单日进行内容检核（检核基数比例、金额、人数等），检核完毕后，点击预发布锁定账单，再发布。这时候财务系统生成应收账单，101前端才会有单子||
|2、 预发布和发布有什么关联/区别|发布的前提为账单是预发布的状态，预发布是为了锁定账单，因为增减员和账单日 有时间差，进行账单锁定，所有账单不会随之变化  <br>（量多的账单日 预发布时间 会有所延长）||
|3、账单组是否能知道账单有变化的|账单组无法知晓账单有变化，唯一的途径是来自于客户需求。  <br>客户通过服务单、400等发起需求到账单组，账单组在分毫系统中点击更新账单  <br>具体流程：  <br>①客户发起101服务单-账单组（手动更新）-客户  <br>②客户通过400/IM/企微发起需求-客成（发起鲲鹏服务单）-账单组（更新）-客成-客户||
|4、更新账单，当账单为支付/申请/开票的状态下 是如何处理的|（1）未支付，未申请  <br>账单组收到需求后，在分毫系统进行撤回，撤回成功，再更新  <br>（2）未支付，已申请  <br>账单组收到需求后，在分享系统进行撤回，撤回失败（已核销/已申请），在财务系统发现没有核销，转财税组核实是否已开票：  <br>①未开票，驳回发票申请后，转账单组在分毫系统更新账单  <br>②已开票（申请先票后款的），财税组告知无法更新账单，账单组终止此需求（若个别商户强烈要求更新，此时需各端联系客户退回发票，走发票废弃流程）  <br>（3）已支付，未申请  <br>账单组收到需求后，在分享系统进行撤回，撤回失败（已核销/已申请），在财务系统撤销核销，转账单组在分毫系统更新账单  <br>（4）已支付，已申请  <br>账单组收到需求后，在分毫系统进行撤回，撤回失败（已核销/已申请），在财务系统撤销核销，发现仍有，转财税组核实是否已开票：  <br>①未开票，财税组驳回发票申请后，转账单组在分毫系统更新账单  <br>②已开票，财税组告知无法更新账单，账单组终止此需求（若个别商户强烈要求更新，此时需各端联系客户退回发票，走发票废弃流程）_注：原已支付的钱会退回至客户的101账户余额内_  <br>系统说明：  <br>分毫系统——进行撤回 更新动作，并判断是否已进入下一状态（已核销/已申请），分毫系统对账单进行撤回，当撤回失败，说明是该账单 已核销/已申请  <br>财务系统——进行撤销 退款动作，并判断是否已进入下一状态（已申请/已出票），财务系统对已核销的账单进行撤销，当撤销失败，说明该账单已申请/已出票  <br>财税组——判断发票状态是已申请/已开具，并对当前状态做相关处理，群内根据账单组提出的无法撤销的账单进行查看，若是已申请但未开票，将申请驳回；若是已开票，告知无法进行账单更新||

### **（七） 平移——待进一步了解**

涉及三种类别进行相关的梳理

“平移”含义：

①仅人事平移，客户的员工由商户A移到商户B，但不做101HR上增减员的处理

②人事+保险平移，客户的员工由商户A移到商户B，且在101HR上做增减员进行参保库的更改

③保险平移，客户的员工所在商户未发生更改，仅进行参保库的更改（员工换库）

关注账单平移，公积金平移

相关操作：商务合同、商户、账单、发票、保险、人事合同、待遇申领
