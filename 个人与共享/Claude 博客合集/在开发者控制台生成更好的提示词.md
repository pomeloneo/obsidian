你现在可以在 Anthropic 控制台中生成生产级提示模板。描述你想要实现的目标，Claude 将使用链式思维推理等提示工程技术来创建有效、精确且可靠的提示。

此功能旨在帮助提示工程新手，同时也为经验丰富的提示工程师节省时间。提供有关任务和期望输出格式的详细信息将获得最佳结果。

虽然生成的提示并不总是产生完美结果，但它们通常优于新手编写的手工提示。生成的提示模板也是可编辑的，让你可以快速调整以获得最佳性能。

## 提示最佳实践

生成的提示模板使用了许多[提示工程最佳实践](https://docs.anthropic.com/en/docs/prompt-engineering)。其中一种是[角色设定](https://docs.anthropic.com/en/docs/give-claude-a-role)，鼓励 Claude 承担所选任务专家的特征。在内容审核示例中：

```jsx
You will be acting as a content moderator to classify chat transcripts as either approved or rejected based on a provided content moderation policy.
```

另一种是[链式思维](https://docs.anthropic.com/en/docs/let-claude-think)推理，给 Claude 在回答前收集思路的时间和空间。当被要求生成基于客户交易历史的产品推荐提示时：

```jsx
In a <scratchpad>, brainstorm 3 different product recommendations you could make to this customer based on their transaction history. For each potential recommendation, provide a brief rationale explaining why you think it would be a good fit for this customer.
```

此外，模板经常将"变量"——可以插入自定义数据的输入字段——放在 [XML 标签](https://docs.anthropic.com/en/docs/use-xml-tags)之间。这遵循了另一个关键最佳实践，通过提供清晰的结构来明确划分提示的不同部分：

```jsx
Your task is to translate a piece of code from another programming language into Python.

Here is the code to translate:

<code>
CODE
</code>

The code is written in LANGUAGE.
```

在某些情况下，你会看到 Claude 编写[示例输入和输出](https://docs.anthropic.com/en/docs/use-examples)来为自己提供关于你想要的答案类型的明确方向。你可以编辑这些示例以匹配你期望的输出格式。

## 幕后原理

提示生成器基于一个长提示，它本身使用了许多已提到的技术：

- 它包含大量任务描述和提示模板的示例，展示 Claude 如何从任务描述转变为提示模板

- 它鼓励 Claude 在编写模板之前规划模板的结构，给 Claude 收集思路的时间

- 它有一个由 XML 标签组成的强"脊柱"，标记每个部分的开始和结束以增强可读性
