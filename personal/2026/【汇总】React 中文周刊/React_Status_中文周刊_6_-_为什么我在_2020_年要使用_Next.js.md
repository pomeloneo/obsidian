## 🔥 本周热门

▶  **为什么我在 2020 年使用 Next.js** — 作者提出一个大胆的主张，认为 “Next.js 是 React 的未来”。视频中，他试图对该观点进行剖析，并解释了他如何看待 React 在服务器端开发扮演的角色。（视频时长：10分钟）

Lee Robinson

**Arwes: 一个未来主义的科幻且赛博朋克的 UI 组件库** — 当使用 React 久了以后，你会发现 React UI 库之间越来越相似，而其中的差异好像只是换了个主题。而这时备受推崇的 Arwes 出现了，因为它非常特别，仿佛把我们带入到了“80年代科幻电影”的感觉。

Romel Pérez

**在 React 中构建简单的 FLIP 动画** — 你的 UI 动画是否存在反应慢且掉帧的问题？本文将教你通过实现 FLIP 动画来避免这个问题。

Travis Arnold

**radEventListener：客户端框架性能的历险记** — 本文是少有的 React 性能剖析案例，详细、客观且数据丰富。更重要的是，其结论可能会导致你以使用 React 以外的库来创建应用。

Jeremy Wagner

▶  **值得学习的全栈课程，使用了 React，GraphQL 以及 TypeScript** — 这是一个非常棒的全栈视频教程，在短短 10 天内超过 10 万次播放，时长约 14 小时。

Ben Awad

## 📘 教程和趣事

**创建你的第一个 React Native Android App** — 本教程将帮助你把 React 技能延伸到移动应用开发领域。然后，你将有机会创造一个 Instagram 杀手级应用。

Esther Vaati

**Storybook 作者亲自为你介绍 Storybook** — 尽管我们最近多次提到了 Storybook，但是在我们看来，本文能更好地指导你开始使用 Storybook，并使用其开发流行的 UI 组件。没人能比 Storybook 作者介绍的更好。

Dominic Nguyen / Tom Coleman

**在 React Native 应用中使用 Mobx 管理状态** — 在看似无休止的状态管理解决方案中，本文选择使用“久经测试”且可扩展的 Mobx 来为 React Native 应用提供状态管理。

Fortune Ikechi

**2020 年开发 React Native 应用前的检查清单** — 飞行员和宇航员必须要使用预检清单，你也应该如此：React Native 应用的“预检”清单。它将确保在开始开发应用之前，你已经了解了所有必要的细节。

Gant Laborde

**Full Stack Serverless：一个使用 GraphQL, CDK, AppSync, 和 React 构建的实时聊天应用** — 本文将通过介绍如何在短短几分钟之内使用 AWS 的 Cloud Developent Kit 和 AppSync，以及 React 和 GraphQL 来部署完整的全栈应用，来向你解释什么是 “full stack serverless”。

Nader Dabit

## 🔧 代码和工具

**Resium：CesiumJS 的 React 版本，开源的 3D 地图库** — Cesium 提供了出色的 3D 地理空间数据展示功能。Resium 提供了一系列 React 组件来实现 Cesium 的功能，可以很方便的集成到你的 React 应用中。

Darwin Education / Eukarya

**Fresh：“极其简单”的 Form 表单组件库** — 尽管表单是个老生常谈的话题了，但 Fresh 仍然可以让你更快速、简单地创建表单。

Level Up Tutorials

**DevExtreme Reactive：深度集成了 Bootstrap 和 Material-UI 的组件库** — Bootstrap 是网站开发中常用的组件库，当然 Material-UI 也不例外，于是有人基于他们写了一套业务组件库，来看下这些组件的演示 demo 吧。

Developer Express Inc.

**D-Tale：Pandas 数据可视化组件库** — 当遇到 Excel 无法完成的工作时，你可能首先想到的是利用 Python 的 Pandas 库来进行更深入的分析。D-Tale 是使用 React 构建的快速分析底层数据结构的 UI 组件库。

Man Group

**Bumbag：构建可访问性友好且主题可定制化的 React 应用** — 以前我们总是说没有足够的时间和精力来使我们的应用更易于访问，而现在有了 Bumbag，它将提高我们应用的可访问性，是基于 Reakit 构建。
