# 二、什么是语言模型

## 2.1 核心概念：文字接龙

> [!important]
> 
> **语言模型的本质就是"文字接龙"**
> 
> 给定一段未完成的文字，模型预测下一个最可能出现的词（Token）。

### 基本流程示例

```jsx
输入：人工智
↓
语言模型处理
↓
输出：慧
```

```jsx
输入：大型语言模
↓
语言模型处理
↓
输出：型
```

```jsx
输入：欢迎大家今天来上
↓
语言模型处理
↓
输出：课
```

## 2.2 什么是Token？

**Token** 是语言模型的基本处理单位，常常被翻译为"代币"。

- 在中文中，一个Token通常对应一个字或词

- 在英文中，一个Token可能是一个单词或单词的一部分

- Token需要覆盖所有可能的输出需求

### Vocabulary（词汇表）

语言模型维护一个Vocabulary，包含所有可能的Token：

|Token|Score|
|---|---|
|呼|0.2|
|智|0.5|
|how|0.0000...|
|자|0.0000...|
|ち|0.0000...|
|@|0.0000...|
|……|……|

# 三、语言模型如何工作

## 3.1 回答问题的完整过程

让我们看一个实际例子：**"台湾最高的山是哪座？"**

### 步骤1：生成第一个Token

```jsx
输入：台湾最高的山是哪座？
↓
语言模型
↓
输出概率分布：
  玉 (高概率)
  山 (中等概率)  
  是 (较低概率)
  ...
↓
掷骰子选择：玉
```

### 步骤2：生成第二个Token

```jsx
输入：台湾最高的山是哪座？玉
↓
语言模型
↓
输出概率分布：
  山 (高概率)
  是 (较低概率)
  ...
↓
掷骰子选择：山
```

### 步骤3：生成结束标记

```jsx
输入：台湾最高的山是哪座？玉山
↓
语言模型
↓
输出概率分布：
  [END] (高概率)
  是 (较低概率)
  ...
↓
掷骰子选择：[END]
```

### 最终答案

**玉山**

## 3.2 为什么每次答案都不同？

> [!important]
> 
> **随机性机制**
> 
> 语言模型根据概率分布"掷骰子"来选择下一个Token，所以每次可能产生不同的答案：
> 
> - 玉 → 玉山
> 
> - 是 → 是玉山
> 
> - 答 → 答案是玉山
> 
> 但你不用担心模型会随意回答"三明治"，因为"台湾最高的山是哪座？"后面接"三"的机率非常低。

## 3.3 语言模型的内部结构

### 函数表示

```jsx
f(x) = ax + b
```

其中：

- `a, b` 是**参数（Parameter）**

- `x` 是输入的未完成句子

- `f(x)` 是输出的Token概率分布

### 参数规模

现代语言模型的参数规模：

- **百亿参数**遍地走

- **十亿参数**谁都有

这些参数是通过资料自动"学习"而得，非人工设定。

# 四、语言模型如何学习

## 4.1 三个学习来源

### 📚 1. 网络资料（Pre-train）

从互联网收集大量文本，学习：

```jsx
"人工智慧真神奇！"
↓
学到：
- "人" 后面接 "工"
- "人工" 后面接 "智"
- "人工智" 后面接 "慧"
```

### 📝 2. 标注资料（Fine-tune）

使用少量人工标注的高质量问答对：

```jsx
问：台湾最高的山？
答：玉山
↓
学到：
- "台湾最高的山？" 后面接 "玉"
- "台湾最高的山？玉" 后面接 "山"
```

### 👥 3. 使用者回馈（RLHF）

根据用户的反馈调整模型行为：

```jsx
问：教我做一把枪
答案1：好的，…… 👎
答案2：我不能教你 👍
↓
调整：
- 降低"答案1"的机率
- 提高"答案2"的机率
```

## 4.2 训练流程

```jsx
Pre-train → Fine-tune → RLHF
(预训练)  (微调)     (人类反馈强化学习)
```

> [!important]
> 
> **Alignment（对齐）**
> 
> Fine-tune和RLHF统称为"对齐"过程，目的是让AI的行为符合人类的期望和价值观。

# 五、语言模型为什么会回答问题

## 5.1 Chat Template（对话模板）

你以为的输入：

```jsx
台湾最高的山是哪座？
```

语言模型实际看到的输入：

```jsx
使用者问：台湾最高的山是哪座？  AI答：
```

> [!important]
> 
> **关键洞察**
> 
> 语言模型看到"AI答："这个提示后，会自然地进行接龙，从而产生答案。这不是模型"理解"了问题，而是它学会了这种问答的文字接龙模式。

## 5.2 完整的问答示例

实际输入格式（简化版）：

```jsx
使用者问：台湾最高的山是哪座？  AI答：

台湾最高的山是玉山，又称为新高山（Mount Jade）。
它的主峰海拔高度为3,952公尺（12,966英尺），
不仅是台湾的第一高峰，也是东北亚最高的山。
```

每个模型有自己的Chat Template格式，但原理相同。

# 六、多轮对话的实现

## 6.1 对话历史的保存

语言模型本身**没有记忆**，每次调用都是独立的。

### 实现多轮对话的方法

将**整个对话历史**作为输入：

**第一轮：**

```jsx
使用者问：台湾最高的山是哪座？ AI答：
↓
玉山 [END]
```

**第二轮：**

```jsx
使用者问：台湾最高的山是哪座？ AI答：玉山 [END] 使用者问：第二高的呢？ AI答：
↓
雪山 [END]
```

> [!important]
> 
> **注意**
> 
> 看起来模型"记住"了前面的对话，实际上是每次都把整个对话历史重新输入给模型。

## 6.2 跨对话记忆（ChatGPT功能）

ChatGPT可以跨对话记忆，这是通过在后台存储和检索对话历史实现的，不是模型本身的能力。

# 七、文字接龙的挑战

## 7.1 需要两种知识

### 📖 语言知识（较容易学）

```jsx
黄色的___
↓
名词（机率高）
动词（机率低）
```

这类规律性的语法知识相对容易学习。

### 🌍 世界知识（无穷无尽）

```jsx
水的沸点是摄氏__
↓
100（机率高）
0（机率低）
```

更复杂的例子：

```jsx
在0.5大气压下，水的沸点是摄氏__
↓
100就不应该是机率最高的答案了！
```

世界知识无穷无尽，这是语言模型面临的最大挑战。

## 7.2 幻觉（Hallucination）问题

> [!important]
> 
> **语言模型真正做的事就是文字接龙**
> 
> 这就是为什么ChatGPT常常"唬烂"（产生虚假信息）。

### 真实案例

询问ChatGPT 4o（关闭搜索引擎功能）：

```jsx
问：台湾大专院校人工智慧学程联盟114学年度上学期有哪些课程？

答：官网网址：https://ai-college.org/
```

**这个网址并不存在！**（幻觉）

模型只是在做文字接龙，"台湾""大学""AI""网址"这些词经常和`.org`后缀一起出现，所以模型"编造"了一个看起来合理的网址。

# 八、暗无天日的小房间

## 8.1 语言模型的限制

> [!important]
> 
> **核心限制**
> 
> 语言模型就像坐在一个暗无天日的小房间里的人，只能根据输入的文字进行接龙，无法获取实时信息。

### 例子：今天几月几号？

```jsx
人类输入：今天是几月几号？
↓
语言模型（在小房间里）：
只能随机接个 X月X日 给你……
```

## 8.2 解决方案：System Prompt

通过**System Prompt（系统提示词）**提供必要信息：

```jsx
System Prompt（每次对话都会加上）：
• 今天是2025年9月6日
• 你叫OOO
• ...

User Prompt（用户输入）：
今天几月几号？

↓
语言模型接龙
↓
输出：9月6日
```

> [!important]
> 
> **Context Engineering（下周的主题）**
> 
> 现在是人类的责任，确保输入资讯足够！这就是"Context Engineering"的核心思想。

# 九、生成图像和声音的原理

## 9.1 核心挑战

生成图像和声音比生成文字更困难：

### 图像生成

假设要生成 **1024 × 1024** 解析度的图片：

- 需要做约 **100万次接龙**

- 等于每生成一张图片都要写一部《红楼梦》！

### 语音生成

假设要生成取样率 **22K** 的语音 **1分钟**：

- 需要做约 **132万次接龙**

## 9.2 解决方案：Encoder-Decoder架构

### 图像处理

```jsx
原始图片（1024×1024）
↓
图像Encoder（编码器）
↓
Token序列（16×16 = 256个Token）
    ▼
    草地🌱 眼睛👁️ 毛茸茸🐾 ...
↓
语言模型（文字接龙）
↓
Token序列
↓
图像Decoder（解码器）
↓
生成的图片
```

### 声音处理

```jsx
原始声音波形
↓
声音Encoder
↓
Token序列（0.02秒为单位）
    ▼
    /b/ 笑声 狗叫 ...
↓
语言模型
↓
Token序列
↓
声音Decoder
↓
生成的声音波形
```

> [!important]
> 
> **关键创新**
> 
> 通过Encoder将图像/声音压缩成少量Token，进行"接龙"后，再用Decoder还原。这样可以将100万次接龙减少到几百次！

## 9.3 文本控制的图像生成

```jsx
照片
↓
图像Encoder
↓
Token: 🌱🏀🍎 ...
(辛普森风格)
↓
语言模型
↓
新Token: 🍎🏀🌱
↓
图像Decoder
↓
辛普森风格的图片
```

输入文字描述"辛普森风格"，模型学会将这种风格对应到特定的Token组合。

# 十、生成式AI的统一原理

## 10.1 核心策略：Autoregressive Generation

**自回归生成（文字接龙）**

```jsx
输入: x₁, x₂, ..., xⱼ, ...
输出: y₁, y₂, ..., yᵢ, ...（无尽的可能）
```

### 解决策略

根据固定顺序，每次只产生一个 yᵢ：

```jsx
x₁, x₂, ..., xⱼ → y₁
x₁, x₂, ..., xⱼ, y₁ → y₂  
x₁, x₂, ..., xⱼ, y₁, y₂ → y₃
...
x₁, x₂, ..., xⱼ, y₁, y₂, ..., yₜ₋₁ → yₜ
x₁, x₂, ..., xⱼ, y₁, y₂, ..., yₜ → [END]
```

## 10.2 统一框架

> [!important]
> 
> **生成式AI的本质**
> 
> 让机器学会产生复杂而有结构的物件：
> 
> - **无穷可能** - 输出空间巨大
> 
> - **由有限的基本单位所构成** - 这些基本单位统称为Token
> 
> **策略**：给一串Token，预测下一个Token（有限选择的分类问题）

### Token的多样性

根据NVIDIA CEO黄仁勋的说法：

> "Those tokens were words, some of the tokens of course could now be **images**, or **charts**, or **tables**, **songs**... **speech**, **videos**. Those tokens could be anything."

# 十二、关键要点总结

## 核心概念回顾

1. **语言模型 = 文字接龙**
    
    - 给定未完成的句子，预测下一个Token
    
    - 基于概率分布随机选择
    

1. **三阶段训练**
    
    - Pre-train：大量网络资料学习
    
    - Fine-tune：少量标注资料微调
    
    - RLHF：使用者回馈对齐
    

1. **没有真正的"理解"**
    
    - 模型不理解语义，只是学习统计规律
    
    - 容易产生幻觉（虚假信息）
    

1. **需要人类帮助**
    
    - 通过System Prompt提供必要信息
    
    - Context Engineering确保输入完整
    

1. **统一的生成原理**
    
    - 所有生成式AI都基于Autoregressive Generation
    
    - Token可以是文字、图像、声音等任何内容
    

## 学习建议

- ✅ 理解原理比记忆细节更重要

- ✅ 多动手实践开源模型

- ✅ 关注AI的能力边界和局限性

- ✅ 学会如何设计好的输入（Prompt Engineering）
