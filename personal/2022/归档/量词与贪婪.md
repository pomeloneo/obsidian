# 量词与贪婪

\#正则表达式

# 贪婪,非贪婪模式,独占模式

贪婪模式: 简单地说就是尽可能进行最长匹配 非贪婪模式: 尽可能进行最短匹配

[![](https://www.notion.so%E9%87%8F%E8%AF%8D%E4%B8%8E%E8%B4%AA%E5%A9%AA/2b03098dcc203c648a40f89a0ba77fc3.png)](https://www.notion.so%E9%87%8F%E8%AF%8D%E4%B8%8E%E8%B4%AA%E5%A9%AA/2b03098dcc203c648a40f89a0ba77fc3.png)

## 贪婪模式(greedy)

表示次数的量词默认是贪婪的 在贪婪模式下会尝试尽可能最大长度去匹配

看一个字符串 aaabb使用 正则(a*)匹配的过程

[![](https://www.notion.so%E9%87%8F%E8%AF%8D%E4%B8%8E%E8%B4%AA%E5%A9%AA/63e5c750b66f6eb914c73befdba43f97.jpg)](https://www.notion.so%E9%87%8F%E8%AF%8D%E4%B8%8E%E8%B4%AA%E5%A9%AA/63e5c750b66f6eb914c73befdba43f97.jpg)

a*在匹配开头的a时, 会尝试尽量匹配更多的a, 直到第一个字母b不满足要求为止 后面的每次匹配都得到了空字符串

## 非贪婪模式(lazy)

在量词的后面加上问号(?) 就开启了非贪婪模式

[![](https://www.notion.so%E9%87%8F%E8%AF%8D%E4%B8%8E%E8%B4%AA%E5%A9%AA/10e40baa1194b17dcc57a089524a37bc.png)](https://www.notion.so%E9%87%8F%E8%AF%8D%E4%B8%8E%E8%B4%AA%E5%A9%AA/10e40baa1194b17dcc57a089524a37bc.png)

```
>>> import re
>>> re.findall(r'a*', 'aaabb')  # 贪婪模式
['aaa', '', '', '']
>>> re.findall(r'a*?', 'aaabb') # 非贪婪模式
['', 'a', '', 'a', '', 'a', '', '', '']
```

## 独占模式(possessive)

不管是贪婪还是非贪婪模式都需要发生回溯才能完成相应的功能 但是在某些场景下不需要回溯, 匹配不上返回失败就可以了 因此正则中还有另一种模式 独占模式 类似于贪婪匹配, 但是匹配过程不会发生回溯

**贪婪模式**

**正则表达式: xy{1,3}z**

**字符串: xyyz**

在匹配时y{1,3}会尽可能长的去匹配, 当匹配完xyy之后, 由于y要求尽可能的匹配最长, 也就是匹配3个 但是字符串后面是个z就会导致匹配不上, 这个时候正则就会向前回溯 吐出当前的字符z, 接着用正则中的z去匹配

[![](https://www.notion.so%E9%87%8F%E8%AF%8D%E4%B8%8E%E8%B4%AA%E5%A9%AA/7a9636b588963e5af9619837fe5a6888.png)](https://www.notion.so%E9%87%8F%E8%AF%8D%E4%B8%8E%E8%B4%AA%E5%A9%AA/7a9636b588963e5af9619837fe5a6888.png)

**非贪婪模式 正则表达式: xy{1,3}?z 字符串: xyyz**

由于y{1,3}?代表匹配1到3个y, 而且是非贪婪模式, 尽可能少的去匹配 当匹配第一个y之后, 也就是在匹配到字符串中的xy之后, 正则会用z和字符串中的xy后面的y比较 发现正则z和y匹配不上, 这时正则会发生向前回溯, 重新查看匹配两个y的情况, 匹配上正则中的xyy, 然后再用z去匹配字符串中的z

[![](https://www.notion.so%E9%87%8F%E8%AF%8D%E4%B8%8E%E8%B4%AA%E5%A9%AA/2177c740a2d5dd805f3157d54636500c.png)](https://www.notion.so%E9%87%8F%E8%AF%8D%E4%B8%8E%E8%B4%AA%E5%A9%AA/2177c740a2d5dd805f3157d54636500c.png)

独占模式和贪婪模式很像, 独占模式会尽可能多地去匹配 如果匹配失败就结束, 不会进行回溯 具体方法就是在量词后面加上加号(+)

**非贪婪模式**

**正则表达式: xy{1,3}+yz**

**字符串: xyyz**

# 总结

正则中量词默认是贪婪匹配 如果想要进行非贪婪匹配需要在量词后面加上问号 贪婪和非贪婪匹配都可能进行回溯 独占模式也是进行贪婪匹配, 但不进行回溯
