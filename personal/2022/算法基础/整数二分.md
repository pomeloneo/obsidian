## 整数二分的性质

- 整数二分的性质不在于是否具有单调性

- 整数二分的性质在于寻找二分的边界

注意这里所说的边界是指**数组的索引**

  

## 二分的解

对于每一次二分来说，都是在寻找更靠近边界的值

虽然对于某些情况来说可能是无解的，但是对于二分搜索的结果来说一定是有解的

  

> [!info] 789. 数的范围 - AcWing题库  
> 给定一个按照升序排列的长度为 $n$ 的整数数组，以及 $q$ 个查询。 对于每个查询，返回一个元素 $k$ 的起始位置和终止位置（位置从 $0$ 开始计数）。 如果数组中不存在该元素，则返回 -1 -1 。 输入格式 第一行包含整数 $n$ 和 $q$，表示数组长度和询问个数。 第二行包含 $n$ 个整数（均在 $1 \sim 10000$ 范围内），表示完整数组。 接下来 $q$ 行，每行包含一个整数 $k$，表示一个询问元素。 输出格式 共 $q$ 行，每行包含两个整数，表示所求元素的起始位置和终止位置。 如果数组中不存在该元素，则返回 -1 -1 。 数据范围 $1 \le n \le 100000$ $1 \le q \le 10000$ $1 \le k \le 10000$ 输入样例： 6 3 1 2 2 3 3 4 3 4 5 输出样例：  
> [https://www.acwing.com/problem/content/791/](https://www.acwing.com/problem/content/791/)  

例如这个题目

假如在 1 2 2 3 3 4 这个序列当中寻找3的左右边界

答案是 3 4

### 二分一定有解

假如需要的数字是2.5，对于题目来说是无解的，因为序列当中本就没有2.5这个元素

但是对于二分来说肯定是有解的

假设寻找2.5的左边界

其实每次的二分都是在查找最靠左的 大于等于2.5的元素位置

其实就是最右面的2（index也是2）

对于查找有边界也是同理

每次二分都是在查找 最靠右的小于等于2.5的元素位置，其实还是最右边的2（index也是2）

  

## 二分的模板

上面提到的题目是比较具有代表性的二分题目

分别查找元素的左右边界

针对左右边界两种情况分别对应模板来处理

- 1

```cpp
int q[];
int x;
int l = 0;
int r = q.size() - 1;

// 查找左边界
while(l < r) {
	int mid = l + r >> 1;
	if (check(q[mid])) {
		r = mid
	} else {
		l = mid + 1
	}
}
```

- 2

```cpp
int q[];
int x;
int l = 0;
int r = q.size() - 1;

while(l < r) {
	// 注意这里的 + 1
	// 如果不 +1 会发生死循环
	int mid = l + r + 1 >> 1;
	if (check(q[mid]) {
		l = mid;
	} else {
		r = mid - 1;
	}
}
```

注意模板当中，计算mid时的 +1操作。
