# 短视频分享到站内私信与站外渠道埋点链路分析

分析时间：2026-05-21（2026-05-22 更新站内私信判断口径）  
代码范围：`/Users/bytedance/gitlab/TikTok`  
关注场景：用户在短视频 Feed 中点击右侧分享按钮，打开分享面板后分享到站内私信或站外渠道。

## 结论

短视频分享链路可以拆成四段：

1. **点击右侧分享按钮**：Feed 侧先上报入口点击，主要是 `click_share_button` 和 `click_more_button`。
2. **拉起分享面板**：通过 `AWEShareService showPanelWithAweme` 创建分享面板，并在 `AWEShareContext.stats` 写入 `enter_from`、`panel_source=share_panel`、`previous_page`、`extraLogInfo` 等上下文。
3. **站内私信**：IM 头像区会先上报曝光和点击，再在发送链路上报 `share_video` 或 `share_video_to_chat`。
4. **站外渠道**：选择 WhatsApp/Facebook/SMS/System/Copy 等平台后走 `AWEShare` task，核心上报 `share_video`；部分平台成功后还会上报 `share_video_success`，系统分享还会补 `share_video_more_track`。

需要注意：`share_video` 不是纯站外事件。站内“一步发送/头像 undo”链路也可能上报 `share_video`，此时 `platform=chat_head`，并带 `conversation_id`、`to_user_id`、`chat_type` 等 IM 参数。

站内私信建议按层拆分口径：

- **点击/选择目标**：看 `share_head_click_total`，表示用户点击了 IM 头像或会话，不等价于发送。
- **发起分享到站内私信**：看 `share_video_to_chat` 或 `share_video(platform=chat_head)`。只看 `share_video_to_chat` 会漏掉头像一步发送/undo 链路；只看 `share_video` 又必须用 `platform` 过滤，否则会混入站外渠道。
- **严格发送成功**：`share_video_to_chat` 和 `share_video(platform=chat_head)` 更接近发起发送口径；如果要看成功，需要继续结合 IM 发送结果、回调或成功状态链路。

## 站内与站外口径对比

| 维度 | 站内私信 | 站外渠道 |
| --- | --- | --- |
| 入口点击 | `click_share_button`、`click_more_button` | 同左 |
| 面板曝光 | `im_share_head_show`、`internal_share_panel_available_user` | 无统一普通 Feed `share_panel_show`，主要看后续渠道 task |
| 点击目标 | `share_head_click_total` | 通常进入 channel task，不单独以这个事件表示 |
| 发送/分享主事件 | 完整发起发送口径：`share_video_to_chat` 或 `share_video(platform=chat_head)` | `share_video(platform=具体平台)` |
| 成功补充 | 严格成功需结合 IM 发送结果、回调或成功状态链路，不能只靠 `share_video_to_chat` | 部分平台补 `share_video_success` |
| 目标信息 | `to_user_id`、`conversation_id`、`chat_type`、`chat_cnt`、`group_chat_cnt` | `platform`、`share_url`、`share_link_id`、`share_form`、`activity_type` |
| 电商字段 | 一步发送的 `share_video` 会补；`share_video_to_chat` 主链路不在当前片段看到同样补法 | `share_video` 明确补 `addECInfoIfValidForVideo` |

## 推荐排查顺序

如果要在日志里还原“点击短视频分享到私信/站外”的完整链路，可以按下面顺序串：

### 站内私信

1. `click_share_button` / `click_more_button`
2. `im_share_head_show`
3. `share_head_click_total`
4. `share_video_to_chat`，或 `share_video` 且要求 `platform=chat_head`
5. 可选：`share_video_info` / `share_video_to_chat_info`
6. 可选：`enter_chat`，通常 `enter_method=share_toast`

### 站外渠道

1. `click_share_button` / `click_more_button`
2. `share_video`，看 `platform` 是否为外部平台或 `copy/system`
3. `share_video_acl`
4. 可选：`share_video_success`
5. 若系统分享：`share_video_more_track`

## 注意事项

- 以上结论基于静态代码分析，线上是否走一步发送、多选发送、统一分享面板、FCP 面板等，会受 AB、场景、内容类型、权限、风控弹窗影响。
- `share_video` 是复用事件，必须结合 `platform` 判断站内/站外：`chat_head` 基本是站内头像发送；WhatsApp/Facebook/SMS/System/Copy 等是站外或外部分享渠道。
- `share_head_click_total` 是点击/选择目标口径，不应单独作为站内私信发送口径；完整发起发送建议合并 `share_video_to_chat` 与 `share_video(platform=chat_head)`。
- `click_more_button` 名字容易误导。当前短视频右侧分享按钮点击也会复用 `trackClickMoreButtonWithTimeStamp`，所以它不只表示“更多按钮”。
