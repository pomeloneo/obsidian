# Q8｜微调：是否将因技术发展变得鸡肋？

[![](https://static001.geekbang.org/resource/image/ab/47/ab0952yy040e9a133ae184039772e247.jpg)](https://static001.geekbang.org/resource/image/ab/47/ab0952yy040e9a133ae184039772e247.jpg)

作者介绍：邢云阳，某大厂 AI 与容器技术专家

Q：MCP 的出现让微调显得鸡肋，因为私有数据都可以通过 MCP 得到，唯一的问题是模型支持的上下文长度是否足够长？

（注：以下内容截取自极客时间专栏《DeepSeek 应用开发实战》）

邢云阳：首先我们要想明白一个点，那就是 MCP 只是一个协议，是为了大家按统一标准做 AI 开发，大家生产的 tool 可以互相无缝衔接调用，所以抛开这个协议，MCP 能干的事情，用 Agent 一样能干。

那清楚这一点后，我们就可以去理解为什么有 Agent 技术了，还要做微调。这是因为 Agent 做得再好，也始终是需要借助外力的。就好比你问我问题，我如果脑子里有（相当于微调），我就可以直接回复你，省时省力，如果我没有，那我就得去百度（相当于 Agent ）。所以，微调肯定比 Agent 效果好，但也比 Agent 难度大很多很多，这也是微调的普及性不如 Agent 的原因。
