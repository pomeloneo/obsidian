# 临床报告写作

## 概述

临床报告撰写是精确、准确并符合监管标准的记录医疗信息的过程。该技能涵盖临床报告的四大类：用于期刊出版的病例报告、用于临床实践的诊断报告、用于监管提交的临床试验报告以及用于医疗记录的患者文档。将此技能应用于医疗保健文档、研究传播和法规遵从性。

**关键原则：临床报告必须准确、完整、客观，并符合适用法规（HIPAA、FDA、ICH-GCP）。** 患者隐私和数据完整性至关重要。所有临床文件必须支持基于证据的决策并符合专业标准。

## 何时使用此技能

该技能应该在以下情况下使用：
- 撰写临床病例报告以提交期刊（CARE指南）
- 创建诊断报告（放射学、病理学、实验室）
- 记录临床试验数据和不良事件
- 准备临床研究报告（CSR）以供监管提交
- 撰写患者进展笔记、SOAP笔记和临床总结
- 起草出院总结、H&P 文件或咨询说明
- 确保 HIPAA 合规性和正确的去识别化
- 验证临床文件的完整性和准确性
- 准备严重不良事件（SAE）报告
- 创建数据安全监控委员会（DSMB）报告

## 核心能力

### 1. 期刊发表的临床病例报告

临床病例报告描述了不寻常的临床表现、新的诊断或罕见的并发症。他们对医学知识做出了贡献，并发表在同行评审的期刊上。

#### CARE 准则合规性

CARE (CAse REport) 指南为案例报告写作提供了标准化框架。所有病例报告均应遵循此清单：

**标题**
- 包含“案例报告”或“案例研究”一词
- 指出关注领域
- 示例：“年轻患者急性心肌梗死的异常表现：病例报告”

**关键字**
- 2-5 个关键字用于索引和搜索
- 尽可能使用 MeSH（医学主题标题）术语

**摘要**（结构化或非结构化，150-250字）
- 简介：此案有何独特之处或新颖之处？
- 患者关注的问题：主要症状和主要病史
- 诊断：初级和次级诊断
- 干预测量指标：关键治疗方法和程序
- 结果：临床结果和随访
- 结论：主要收获或临床教训

**简介**
- 医疗状况的简要背景
- 为什么这个案例新颖或重要
- 类似案例文献综述（简述）
- 是什么让这个案例值得报道

**患者信息**
- 人口统计（年龄、性别，race/ethnicity（如果相关））
- 病史、家族史、社会史
- 相关合并症
- **去识别化**：删除或更改18个HIPAA标识符
- **患者同意**：记录发布知情同意书

**临床发现**
- 主诉和出现症状
- 体检结果
- 症状时间线（考虑时间线图或表格）
- 相关临床观察

**时间线**
- 关键事件的时间总结
- 症状、诊断、干预测量指标、结果的日期
- 可以以表格或图形的形式呈现
- 示例格式：
  - 第 0 天：最初出现症状 X、Y、Z
  - 第 2 天：进行诊断测试 A，发现结果 B
  - 第 5 天：用药物 C 开始治疗
  - 第 14 天：观察到临床改善
  - 第 3 个月：后续检查显示完全解决

**诊断评估**
- 进行的诊断测试（实验室、成像、程序）
- 结果和解释
- 考虑鉴别诊断
- 最终诊断的基本原理
- 诊断挑战

**治疗干预**
- 药物（名称、剂量、途径、持续时间）
- 进行的程序或手术
- 非药物干预
- 治疗选择的推理
- 考虑替代治疗

**后续行动和结果**
- 临床结果（解决、改善、不变、恶化）
- 随访持续时间和频率
- 长期成果（如果有）
- 患者报告的结果
- 坚持治疗

**讨论**
- 案例的优点和新颖性
- 此案例与现有文献相比如何
- 案例报告的局限性
- 潜在机制或解释
- 临床意义和经验教训
- 未解答的问题或未来研究的领域

**患者视角**（可选但鼓励）
- 患者的经历和观点
- 对生活质量的影响
- 患者报告的结果
- 来自患者的引用（如果适用）

**知情同意**
- 记录患者同意发表的声明
- 如果患者已死亡或无法同意，请描述代理同意书
- 对于儿科病例，parental/guardian同意
- 示例：“已从患者处获得书面知情同意书，以发表本病例报告和随附图像。书面同意书的副本可供本期刊主编审阅。”

详细CARE指南，请参阅`references/case_report_guidelines.md`。

#### 期刊特定要求

不同的期刊有特定的格式要求：
- 字数限制（通常为 1500-3000 字）
- 允许的figures/tables数量
- 参考样式（AMA、Vancouver、APA）
- 结构化摘要与非结构化摘要
- 补充材料政策

在提交之前检查作者的期刊说明。

#### 去识别化和隐私

**18 HIPAA 要删除或更改的标识符**：
1. 名字
2. 小于州的地理分区
3. 日期（年份除外）
4. 电话号码
5. 传真号码
6. 电子邮件地址
7. 社会安全号码
8. 医疗记录号码
9. 健康计划受益人号码
10. 帐号
11. Certificate/license 数字
12. 车辆标识符和序列号
13. 设备标识符和序列号
14. 网页 URLs
15. IP 地址
16. 生物识别标识符
17. 全脸照片
18. 任何其他独特的识别特征

**最佳实践**：
- 使用“患者”而不是名字
- 报告年龄范围（e.g.，“60 多岁的女性”）或确切年龄（如果相关）
- 使用大致日期或时间间隔（e.g.，“3 个月前”）
- 除非必要，否则删除机构名称
- 模糊或裁剪图像中的识别特征
- 获得任何潜在识别信息的明确同意

### 2. 临床诊断报告

诊断报告传达影像学研究、病理检查和实验室测试的结果。它们必须清晰、准确且可操作。

#### 放射学报告

放射学报告遵循标准化结构，以确保清晰度和完整性。

**标准结构**：

**1。患者人口统计**
- 患者姓名（或在研究环境中为 ID）
- 出生日期或年龄
- 病历号
- 考试日期和时间

**2。临床适应症**
- 考试原因
- 相关临床病史
- 有待回答的具体临床问题
- 示例：“排除急性呼吸困难患者的肺栓塞”

**3。技术**
- 成像方式（X 射线、CT、MRI、超声波、PET 等）
- 检查的解剖区域
- 造影剂管理（类型、路线、体积）
- 使用的协议或序列
- 技术质量和限制
- 示例：“使用 100 mL 静脉注射碘造影剂对胸部、腹部和骨盆进行造影增强 CT。未给予口服造影剂。”

**4。比较**
- 之前的影像学研究可供比较
- 先前研究的日期
- 稳定性或与先前成像的变化
- 示例：“比较：来自[日期]的CT箱子”

**5。调查结果**
- 影像学表现的系统描述
- 逐个器官或逐个区域的方法
- 首先是积极的发现，然后是相关的消极发现
- 病变或异常的测量
- 使用标准化术语（ACR词典，RadLex）
- 示例：
  - 肺：双侧磨玻璃样混浊，主要出现在下叶。无实变或胸腔积液。
  - 纵隔：无淋巴结肿大。心脏大小正常。
  - 腹部：肝、脾、胰未见异常。无游离液体。

**6. Impression/Conclusion**
- 主要发现的简要总结
- 临床问题的答案
- 鉴别诊断（如果适用）
- 后续或额外研究的建议
- 怀疑程度或诊断确定性
- 示例：
  - “1.双侧毛玻璃样混浊符合病毒性肺炎或非典型感染。不能排除COVID-19。建议临床相关。
  - 2.没有肺栓塞的证据。
  - 3. 建议在 4-6 周内进行后续成像以评估分辨率。”

**结构化报告**：

许多放射科使用结构化报告模板进行常见检查：
- 肺结节报告（Lung-RADS）
- 乳房成像 (BI-RADS)
- 肝脏成像 (LI-RADS)
- 前列腺成像 (PI-RADS)
- CT结肠造影 (C-RADS)

结构化报告可提高一致性、减少歧义并促进数据提取。

有关放射学报告标准，请参阅`references/diagnostic_reports_standards.md`。

#### 病理报告

病理学报告记录组织标本的显微发现并提供诊断结论。

**外科病理学报告结构**：

**1。患者信息**
- 患者姓名和标识符
- 出生日期、年龄、性别
- 预约医生
- 病历号
- 样本收到日期

**2。样本信息**
- 标本类型（活检、切除、切除）
- 解剖部位
- 偏侧性（如果适用）
- specimens/blocks/slides 的数量
- 示例：“皮肤，左前臂，切除活检”

**3。临床病史**
- 相关临床信息
- 活检指征
- 先前诊断
- 示例：“黑色素瘤病史。新的色素病变，排除复发。”

**4。总体描述**
- 标本宏观外观
- 尺寸、重量、颜色、稠度
- 方向标记（如果存在）
- 切片和取样方法
- 示例：“样本由尺寸为 2.5 x 1.0 x 0.5 cm 的椭圆形皮肤组成。表面上存在直径为 0.6 cm 的色素病变。样本被连续切片并完全放入包埋盒中提交A1-A3。”

**5。微观描述**
- 组织学发现
- 蜂窝特性
- 架构模式
- 存在恶性肿瘤
- 边距（如果适用）
- 特殊染色或免疫组织化学结果

**6。诊断**
- 初步诊断
- 年级和阶段（如果适用）（癌症）
- 保证金状态
- 淋巴结状态（如果适用）
- 癌症概要报告（CAP 方案）
- 示例：
  - "MALIGNANT MELANOMA, SUPERFICIAL SPREADING TYPE
  - 布雷斯洛厚度：1.2毫米
  - 克拉克等级：IV
  - 有丝分裂率：3/mm²
  - 溃疡：不存在
  - 边距：负（最接近的边距0.4厘米）
  - 淋巴血管侵犯：未识别”

**7。评论**（如果需要）
- 附加上下文或解释
- 鉴别诊断
- 额外研究的建议
- 临床相关建议

**天气报告**：

美国病理学家学院 (CAP) 提供癌症标本的概要报告模板。这些检查表确保记录所有相关的诊断要素。

癌症报告的关键要素：
- 肿瘤部位
- 肿瘤大小
- 组织学类型
- 组织学等级
- 入侵程度
- 淋巴血管侵犯
- 神经周围侵犯
- 边距
- 淋巴结（检查数量，阳性数量）
- 病理阶段（TNM分类）
- 辅助研究（分子标记、生物标记）

#### 实验室报告

实验室报告传达临床样本（血液、尿液、组织等）的测试结果。

**标准组件**：

**1。患者和样本信息**
- 患者标识符
- 样本类型（血液、血清、尿液、CSF等）
- 收集日期和时间
- 接收日期和时间
- 订购提供商

**2。测试名称和方法**
- 完整测试名称
- 方法论（免疫分析、分光光度法、PCR等）
- 实验室登录号

**3。结果**
- 定量或定性结果
- 测量单位
- 参考范围（正常值）
- 异常值标志（H = 高，L = 低）
- 突出显示的关键值
- 示例：
  - 血红蛋白：8.5 g/dL (L) [参考：12.0-16.0 g/dL]
  - 白细胞计数：15.2 x10³/μL (H) [参考：4.5-11.0 x10³/μL]

**4。解释**（如适用）
- 结果的临床意义
- 建议的后续或额外测试
- 与诊断的相关性
- 药物水平和治疗范围

**5。质量控制信息**
- 样本充足性
- 标本质量问题（溶血、血脂、凝块）
- 处理延迟
- 技术限制

**临界值报告**：
- 危及生命的结果需要立即通知
- 示例：葡萄糖 <40 或 >500 mg/dL，钾 <2.5 或 >6.5 mEq/L
- 文件通知时间及收件人

有关实验室标准和术语，请参阅`references/diagnostic_reports_standards.md`。

### 3. 临床试验报告

临床试验报告记录了临床研究的实施、结果和安全性。这些报告对于监管提交和科学出版至关重要。

#### 严重不良事件（SAE）报告

SAE 报告记录了临床试验期间意外的严重不良反应。监管要求要求及时向IRBs、赞助商和监管机构报告。

**严重不良事件的定义**：
如果出现以下情况，则不良事件是严重的：
- 导致死亡
- 有生命危险
- 需要住院治疗或延长现有住院治疗时间
- 结果持续或显着disability/incapacity
- 是先天性anomaly/birth缺陷
- 需要干预以防止永久性损伤或损坏

**SAE 报告组成部分**：

**1。学习信息**
- 协议编号和标题
- 学习阶段
- 赞助商名称
- 首席研究员
- IND/IDE 号码（如果适用）
- 临床试验注册号（NCT号）

**2。患者信息（去识别化）**
- 受试者ID 或随机编号
- 年龄，性别，race/ethnicity
- 研究组或治疗组
- 知情同意日期
- 首次研究干预的日期

**3。活动信息**
- 事件描述（叙述）
- 发病日期
- 决议（或正在进行）的日期
- 严重程度（轻度、中度、重度）
- 符合严重性标准
- 结果（已恢复、正在恢复、未恢复、致命、未知）

**4。因果关系评估**
- 与研究干预的关系（不相关、不太可能、可能、很可能、确定）
- 与研究程序的关系
- 与基础疾病的关系
- 因果关系确定的基本原理

**5。采取的行动**
- 研究干预的修改（减少剂量、暂时停止、永久停止）
- 联合用药或治疗
- 住院详情
- 结果和后续计划

**6。预期**
- 根据方案或研究者手册预期
- 需要加急报告的意外事件
- 与已知安全状况的比较

**7。叙述**
- 活动详细描述
- 事件时间表
- 临床课程和管理
- 实验室和诊断测试结果
- 最终诊断或结论

**8。记者信息**
- 记者姓名及联系方式
- 报告日期
- 签名

**监管时间表**：
- 致命或危及生命的意外SAEs：初步报告7天，完整报告15天
- 其他严重突发事件：15天
- IRB 通知：根据机构政策，通常在 5-10 天内

有关详细的SAE报告指南，请参阅`references/clinical_trial_reporting.md`。

#### 临床研究报告(CSR)

临床研究报告是总结临床试验的设计、实施和结果的综合文件。它们作为药物批准申请的一部分提交给监管机构。

**ICH-E3 结构**：

ICH E3 指南定义了临床研究报告的结构和内容。

**主要部分**：

**1。扉页**
- 研究标题和方案编号
- 赞助商和研究者信息
- 报告日期和版本

**2。概要**（5-15 页）
- 整个研究的简要总结
- 目标、方法、结果、结论
- 主要功效和安全性发现

#### 协议偏差

方案偏差是指偏离已批准的研究方案。它们必须被记录、评估和报告。

**类别**：
- **轻微偏差**：不会显着影响患者安全或数据完整性
- **重大偏差**：可能影响患者安全、数据完整性或研究行为
- **违规**：严重偏差，需要立即采取行动和报告

**文件要求**：
- 偏差描述
- 发生日期
- 受影响对象ID
- 对安全和数据的影响
- 纠正和预防测量指标 (CAPA)
- 根本原因分析
- 已实施预防测量指标

### 4. 患者临床文件

患者文档记录临床经历、进展和护理计划。准确的文档支持护理、计费和法律保护的连续性。

#### SOAP 注释

SOAP 笔记是临床实践中最常见的进度笔记格式。

**结构**：

**S - 主观**
- 患者报告的症状和担忧
- 现病史(HPI)
- 与访问相关的系统审查 (ROS)
- 患者自己的话（有帮助时使用引号）
- 示例：“患者报告过去 3 天内呼吸短促恶化，特别是在用力时。否认胸痛、发烧或咳嗽。”

**O - 目标**
- 可测量的临床结果
- 生命体征（体温、血压、心率、呼吸频率、血氧饱和度）
- 体检结果（系统整理）
- 实验室和成像结果
- 示例：
  - 生命体征：T 98.6°F、BP 142/88、HR 92、RR 22、SpO2 91%（室内空气）
  - 一般：轻度呼吸窘迫
  - 心血管：心律正常，无杂音
  - 肺部：双侧基底有爆裂音
  - 四肢：双侧凹陷性水肿 2+

**A - 评估**
- 临床印象或诊断
- 鉴别诊断
- 严重性和稳定性
- 治疗目标的进展
- 示例：
  - “1.急性失代偿性心力衰竭，NYHA类别III
  - 2.高血压，控制不佳
  - 3.慢性肾病，第3阶段”

**P - 计划**
- 诊断计划（进一步测试）
- 治疗计划（药物、手术）
- 患者教育和咨询
- 后续安排
- 示例：
  - “诊断：BNP，胸部X光检查，超声心动图
  - 治疗：将速尿增加至 40 mg PO BID，继续每天服用赖诺普利 10 mg，严格限制液体摄入量至 1.5 L/day
  - 教育：心力衰竭恶化的迹象、每日体重
  - 后续：1 周内预约心脏病科，如果 1 天内体重增加 >2 磅，请致电”

**文档提示**：
- 简洁但完整
- 使用标准医学缩写
- 记录遭遇时间
- 在所有笔记上签名并注明日期
- 避免猜测或判断
- 记录医疗必要性以进行计费
- 包括患者对治疗的反应

对于SOAP笔记模板和示例，请参阅`assets/soap_note_template.md`。

#### 历史和身体 (H&P)

H&P 是在入院或初次接触时进行的综合评估。

**组件**：

**1。主诉 (CC)**
- 简要说明患者寻求护理的原因
- 用患者自己的话
- 示例：“胸痛 2 小时”

**2。现病史 (HPI)**
- 当前问题的详细时间顺序叙述
- 使用OPQRST助记符来表示疼痛：
  - 发病：什么时候开始的？
  - Provocation/Palliation：是什么让它变得更好或更糟？
  - 质量：感觉如何？
  - Region/Radiation：在哪里？它会传播吗？
  - 严重性：有多严重（0-10 等级）？
  - 计时：持续还是间歇？期间？
- 相关症状
- 之前的评估或治疗

**3。既往病史 (PMH)**
- 慢性疾病
- 之前住院经历
- 手术和程序
- 示例：“高血压（2015 年诊断）、2 型糖尿病（2018 年诊断）、既往阑尾切除术（2010 年）”

**4。药物**
- 当前药物的剂量和频率
- 非处方药
- 草药补充剂
- 过敏和反应

**5。过敏**
- 药物过敏及其反应类型
- 食物过敏
- 环境过敏
- 示例：“青霉素（皮疹）、贝类（过敏反应）”

**6。家族史 (FH)**
- 一级亲属的医疗状况
- 父母的年龄和死因
- 遗传性疾病
- 示例：“父亲患有冠状动脉疾病（MI，55 岁），母亲患有乳腺癌（诊断年龄 62）”

**7。社会历史 (SH)**
- 烟草使用（包年）
- 饮酒（每周饮酒）
- 非法药物使用
- 职业
- 居住情况
- 性史（如果相关）
- 示例：“以前吸烟，5 年前戒烟（20 包年历史）。偶尔喝酒（2-3 drinks/week）。担任会计师。与配偶住在一起。”

**8。系统审查 (ROS)**
- 按器官系统对症状进行系统审查
- 通常 10-14 个系统
- 相关的积极和消极
- 系统：体质、眼睛、ENT、心血管、呼吸、GI、GU、肌肉骨骼、皮肤、神经、精神、内分泌、Hematologic/Lymphatic、Allergic/Immunologic

**9。体格检查**
- 生命体征
- 一般外观
- 器官系统的系统检查
- HEENT，颈部，心血管，肺，腹部，四肢，神经，皮肤
- 使用标准术语和缩写

**10。评估和计划**
- 问题列表，其中包含每个问题的评估和计划
- 编号列表格式
- 诊断和治疗计划
- 处置（入院、出院、转院）

对于 H&P 模板，请参阅`assets/history_physical_template.md`。

#### 出院小结

出院总结记录住院情况并向门诊提供者传达护理计划。

**必需元素**：

**1。患者身份识别**
- 姓名、出生日期、病历号码
- 入院和出院日期
- 主治医师
- 入院和出院诊断

**2。住院原因**
- 所呈现问题的简要描述
- 主诉

**3。医院课程**
- 关键事件的时间顺序叙述
- 重要发现和程序
- 对治疗的反应
- 并发症
- 已获得咨询
- 按问题或按时间顺序组织

**4。出院诊断**
- 初步诊断
- 二次诊断
- 并发症
- 合并症

**5。执行的程序**
- 手术
- 侵入性手术
- 诊断程序

**6。排出药物**
- 完整的药物清单和说明
- 入院药物的变化
- 新药及适应症

**7。放电条件**
- 稳定、改进、不变、过期
- 功能状态
- 精神状态

**8。排放处置**
- 家庭、专业护理设施、康复、临终关怀
- 有或没有服务

**9。后续计划**
- 已安排预约
- 推荐后续时间
- 待测试或研究
- 推荐

**10。患者须知**
- 活动限制
- 饮食限制
- 伤口护理
- 寻求护理的警告信号
- 用药说明

**最佳实践**：
- 出院后 24-48 小时内完成
- 对门诊服务提供者使用清晰的语言
- 突出显示重要的待决结果
- 文档代码状态讨论
- 包括提供的患者教育

有关出院摘要模板，请参阅`assets/discharge_summary_template.md`。

## 监管合规性和隐私

### HIPAA 合规性

《健康保险流通与责任法案》(HIPAA) 强制要求保护患者健康信息。

**关键要求**：
- 最低限度的必要披露
- 患者授权使用超过treatment/payment/operations
- 安全存储和传输
- 电子记录的审计追踪
- 违规通知程序

**去识别方法**：
1. **安全港方法**：删除18个标识符
2. **专家判定**：统计方法确认重新识别风险较低

**业务伙伴协议**：
当PHI与第三方共享服务时需要

## 医学术语和标准

### 标准化命名法

**SNOMED CT（系统化医学命名法 - 临床术语）**
- 综合临床术语
- 用于电子健康记录
- 实现语义互操作性

**LOINC（逻辑观察标识符名称和代码）**
- 实验室和临床观察标准
- 促进数据交换和报告

**ICD-10-CM（国际疾病分类，第十次修订，临床修改）**
- 用于计费和流行病学的诊断编码
- 需要报销

**CPT（当前程序术语）**
- 计费过程编码
- 维护者AMA

### 缩写标准

**可接受的缩写**：
使用标准缩写来提高效率，同时保持清晰度。

**请勿使用名单（联合委员会）**：
- U（单位）-写“单位”
- IU（国际单位）-写“国际单位”
- QD、QOD（每天、每隔一天）- 写“每天”或“每隔一天”
- 尾随零 (X.0 mg) - 切勿在小数点后使用
- 缺少前导零 (.X mg) - 始终在小数点之前使用 (0.X mg)
- MS、MSO4、MgSO4 - 写“硫酸吗啡”或“硫酸镁”

全面的术语标准，请参见`references/medical_terminology.md`。

## 临床报告撰写工作流程

### 病例报告工作流程

**第 1 阶段：病例识别和同意（第 1 周）**
- 识别新颖或教育案例
- 获得患者知情同意
- 去识别患者信息
- 收集临床数据和图像

**第 2 阶段：文献综述（第 1-2 周）**
- 搜索类似案例
- 回顾相关病理生理学
- 确定知识差距
- 确定新颖性和重要性

**第 3 阶段：起草（第 2-3 周）**
- 按照CARE指南编写结构化大纲
- 起草所有部分（通过讨论进行摘要）
- 创建时间线和图表
- 格式参考

**第 4 阶段：内部审查（第 3-4 周）**
- 共同作者评论
- 主治医师审查
- 机构审查（如果需要）
- 耐心审查去识别化的草稿

**阶段 5：期刊选择和提交（第 4-5 周）**
- 选择合适的期刊
- 每个期刊指南的格式
- 准备求职信
- 提交稿件

**阶段 6：修订（可变）**
- 回复同行评审员的评论
- 修改稿件
- 重新提交

### 诊断报告工作流程

**实时工作流程**：
- 审查临床适应症和先前的研究
- 解释影像学、病理学或实验室结果
- 使用结构化格式听写或输入报告
- 复杂案件的同行评审
- 最终签出和分发
- 临界值通知（如果适用）

**周转时间 Time 基准**：
- STAT 报告：<1 小时
- 例行报告：24-48小时
- 复杂案件：2-5天
- 待进一步研究：有记录的延迟

### 临床试验报告工作流程

**SAE 报告：24小时至15天**
- 站点确定的事件
- 初步评估和文档
- 因果关系和预期确定
- 报告完成和审核
- 提交给赞助商，IRB，FDA（根据需要）
- 后续报告直至解决

**CSR：学习完成后6-12个月**
- 数据库锁定和数据清理
- 每SAP的统计分析
- 医学作家起草
- 生物统计学家和临床团队审查
- 质量控制审查
- 最终批准和监管提交

## 要避免的常见陷阱

### 病例报告
- **侵犯隐私**：去身份识别不充分或缺少同意
- **缺乏新颖性**：报告常见或有据可查的案例
- **细节不足**：缺少关键临床信息
- **文献综述不佳**：未能在现有知识中融入背景
- **过度概括**：从单个案例中得出广泛的结论

### 诊断报告
- **含糊的语言**：使用不明确的术语，例如“不起眼”，但没有具体说明
- **不完整的比较**：未审查先前的成像
- **缺少临床相关性**：未能回答临床问题
- **技术术语**：过度使用术语而没有解释
- **Delayed 临界值通知**：未传达紧急发现

### 临床试验报告
- **延迟报告**：错过SAE报告的监管截止日期
- **不完整的因果关系**：因果关系评估不充分
- **数据不一致**：数据源之间的差异
- **协议偏差**：未报告或记录不充分的偏差
- **选择性报告**：忽略负面或不利的结果

### 患者文档
- **字迹不清**：纸质记录中的字迹不佳
- **复制转发错误**：传播过时的信息
- **细节不足**：影响计费的模糊或不完整的文档
- **缺乏医疗必要性**：未记录服务适应症
- **缺少签名**：未签名或未注明日期的注释

## 最终清单

在最终确定任何临床报告之前，请验证：

- [ ] 所有必填部分已完成
- [ ] 患者隐私保护（HIPAA合规性）
- [ ] 已获得知情同意（如果适用）
- [ ] 准确且经过验证的临床数据
- [ ] 适当的医学术语和编码
- [ ] 清晰、专业的语言
- [ ] 根据指南正确格式化
- [ ] 适当引用的参考文献
- [ ] 正确标记的数字和表格
- [ ] 拼写检查和校对
- [ ] 满足监管要求
- [ ] 遵循机构政策
- [ ] 签名和日期
- [ ] 质量保证审查已完成

---

**最后说明**：临床报告写作需要注意细节、医疗准确性、法规遵从性和清晰的沟通。无论是记录患者护理、报告研究结果还是传达诊断结果，临床报告的质量都直接影响患者安全、医疗保健服务和医学知识进步。在所有临床文档中始终优先考虑准确性、隐私性和专业性。
