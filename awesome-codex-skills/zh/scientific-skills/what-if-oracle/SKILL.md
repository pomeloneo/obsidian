# 假设预言机 — 可能性空间探索者

一个通过严格的多分支场景分析探索不确定未来的结构化系统。预言机不是进行单一预测，而是映射了完整的**可能性空间**——分支时间线，其中每条路径都有自己的逻辑、概率和后果。

基于假设陈述范式：推测性问题（“如果 X 会怎样？”）不是无聊的白日梦，而是一种**基本的计算操作**——在将资源投入到未来之前，大脑模拟未来的方式。

已发表的研究：[假设陈述 (DOI: 10.5281/zenodo.18736841)](https://doi.org/10.5281/zenodo.18736841) | [IDNA 整合 v2 (DOI: 10.5281/zenodo.18807387)](https://doi.org/10.5281/zenodo.18807387)

## 核心原理：0·IF·1

每个场景分析都包含三个要素：

- **0** — 未表达的状态（尚未发生的事情，潜力）
- **1** — 表达的状态（是什么，当前的现实）
- **IF** — 条件键（将 0 转变为 1 的决定、事件或变化）

分析的质量取决于 IF 的精度。模糊的“如果出了问题怎么办？”产生模糊的结果。一个精确的“如果我们的主要供应商在第三季度将价格提高 30% 怎么办？”产生可操作的情报。

## 如何运行Oracle

### 第一阶段——提出问题

提出用户的假设问题并加以强化：

**分解为组件：**

- **变量：**具体发生了什么变化？ （每次分析一个变量）
- **幅度：** 多少？ （如果可能的话，量化）
- **时间范围：** 多长时间？
- **上下文：**更改之前的当前状态是什么？

**如果问题含糊不清，请明确它：**

- “如果AI接管怎么办？” →“如果[特定行业]当前 40% 的知识工作任务在 3 年内由AI实现自动化，会怎样？”
- “如果我们失败了怎么办？” →“如果从现在开始连续 6 个月月收入低于 5,000 美元怎么办？”

在继续之前，向用户提出尖锐的问题以供确认。

### 第 2 阶段 — 绘制可能性空间

使用此框架生成 **4-6 个场景分支**：

| 分公司             | 定义                                                                   | 目的                                            |
| ------------------ | ---------------------------------------------------------------------------- | -------------------------------------------------- |
| **Ω 最佳情况**    | 一切顺利。关键假设全部成立。幸运的突破发生了。     | 定义上限——最大的上升空间是多少？    |
| **α 可能的情况**  | 给定当前证据的最可能路径。没有什么大的惊喜。               | 将期望锚定在现实中                     |
| **Δ 最坏情况**   | 关键假设失败。有两件事同时出错。                    | 定义下限——最大的缺点是什么？    |
| **Ψ 通配符**    | 一个无人追踪的意外变量出现了。黑天鹅领地。 | 难以想象的压力测试                   |
| **Φ 逆势**   | 事实证明，与共识观点相反的观点是正确的。                     | 挑战群体思维并揭示隐藏的假设 |
| **无穷二阶** | 一阶效应引发了无人预料到的级联后果。     | 绘制连锁反应图                             |

### 第四阶段——合成

分析所有分支后，提供：

**概率分布：**

```
Ω Best Case ····· [██████░░░░] 15%
α Likely Case ··· [████████░░] 45%
Δ Worst Case ···· [██████░░░░] 20%
Ψ Wild Card ····· [███░░░░░░░]  8%
Φ Contrarian ···· [████░░░░░░]  7%
∞ Second Order ·· [███░░░░░░░]  5%
```

**强有力的行动：** 哪些行动对多个分支机构有益？这些都是无悔的举动——无论未来会发生什么，都要这样做。

**对冲行动：** 哪些准备工作可以在不牺牲上行空间的情况下防范最糟糕的分支？

**决策触发因素：** 哪些具体的、可观察的信号应该促使您更新哪个分支最有可能？定义绊线。

**1% 的洞察力：** 在这种情况下，几乎每个分析它的人都会忽略的一件事是什么？不明显的模式、隐藏的假设、被忽视的变量。

## 黄金比例加权

当证据存在时，使用黄金比例权衡主要场景：

- **主要未来（最有可能）：** 61.8% 的注意力/资源
- **另类未来：** 38.2% 的注意力/资源

这既可以防止对单一路径的过度投入，也可以防止过多的意外事件的稀释。 Nature 使用此比率进行分支（树木、河流、血管）。战略规划也可以。

## 模式

### 快速预言（2-3 分钟）

仅 3 个分支：最佳、可能、最差。简短的叙述。用于快速决策。

### 深度预言（5-10 分钟）

全部6家分行。全面分析后果、触发因素和综合。用于高风险决策。

### 场景链

将一项 Oracle 分析的输出输入到另一项分析中。 “如果分支 Δ 发生，该分支内的可能性是什么？”复杂战略规划的递归深度。

### 逆向预言机

从期望的结果开始并向后推算：“X 发生必须满足哪些条件？实现该结果最可能的路径是什么？”对于目标设定和策略设计很有用。

### 具有竞争力的甲骨文

从多个利益相关者的角度分析相同的假设：“如果我们推出这个产品，从我们的角度、他们的角度和市场的角度来看，可能性空间是什么样的？”

## 这不是什么

- 不是预测——而是可能性图。甲骨文并没有声称自己知道未来；而是声称他知道未来。它可以帮助您为多种未来做好准备。
- 不是水晶球——概率是基于现有证据的估计，而不是确定性。
- 不能替代行动——如果没有后续的决策和执行，世界上最好的情景分析就毫无价值。
