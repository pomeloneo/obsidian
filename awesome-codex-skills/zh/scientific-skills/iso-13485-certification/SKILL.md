# ISO 13485认证文档助理

## 概述

该技能帮助医疗器械制造商为ISO 13485:2016 认证准备全面的文档。它提供了用于创建、审查和差距分析所有必需的质量管理体系 (QMS) 文档的工具、模板、参考和指南。

**该技能提供什么：**
- 现有文档的差距分析
- 所有强制性文档的模板
- 全面的需求指导
- 分步文档创建
- 识别缺失文档
- 合规性检查表

**何时使用此技能：**
- 启动ISO 13485认证流程
- 针对ISO 13485进行差距分析
- 创建或更新QMS文档
- 准备认证审核
- 从FDA QSR过渡到 QMSR
- 与EU MDR要求

## 核心工作流程

### 1. 评估当前状态（差距分析）

**何时开始：** 用户拥有现有文档并需要识别差距

**流程：**

1. **收集现有文档：**
- 要求用户提供当前QMS文档的目录
- 文档可以是任何格式（.txt、.md、.doc、.docx、.pdf）
- 包括任何程序、手册、工作指示、表格

2. **运行间隙分析脚本：**
   ```bash
   python scripts/gap_analyzer.py --docs-dir <path_to_docs> --output gap-report.json
   ```

3. **审核结果：**
- 确定存在 31 个必需程序中的哪一个
- 确定缺失的关键文档（质量手册、MDF 等）
- 计算合规百分比
- 确定缺失文档的优先级

4. **向用户呈现结果：**
-总结现有内容
- 清楚列出缺少的内容
- 提供优先行动计划
- 估计所需工作量

**输出：** 包含优先行动项目的全面差距分析报告

### 2. 了解要求（参考咨询）

**何时使用：** 用户需要了解具体的ISO 13485要求

**可用参考：**
- `references/iso-13485-requirements.md`- 完整的逐条细目
- `references/mandatory-documents.md`- 解释了所有 31 个所需程序
- `references/gap-analysis-checklist.md`- 详细的合规性检查表
- `references/quality-manual-guide.md`- 如何创建质量手册

**如何使用：**

1. **针对具体条款问题：**
- 阅读`iso-13485-requirements.md`中的相关部分
- 用通俗易懂的语言解释要求
- 提供实际示例

2. **对于文件要求：**
- 咨询`mandatory-documents.md`
- 解释必须记录的内容
- 澄清文件何时适用与排除

3. **对于实施指导：**
- 对政策级文件使用`quality-manual-guide.md`
- 提供逐步创建过程
- 显示良好与不良实施的示例

**需要了解的关键参考部分：**

- **第 4 条：**QMS要求、文档、风险管理、软件验证
- **第 5 条：** 管理责任、质量政策、目标、管理评审
- **第 6 条：** 资源、能力、培训、基础设施
- **第 7 条：** 产品实现、设计、采购、生产、可追溯性
- **第 8 条：** 测量、审计、CAPA、投诉、数据分析

### 3. 创建文档（基于模板生成）

**何时使用：** 用户需要创建特定的QMS文档

**可用模板：**
- 质量手册：`assets/templates/quality-manual-template.md`
- CAPA程序：`assets/templates/procedures/CAPA-procedure-template.md`
- 文档控制：`assets/templates/procedures/document-control-procedure-template.md`

**文档创建流程：**

1. **确定需要创建的内容：**
- 基于差距分析或用户请求
- 首先确定关键文档的优先级（质量手册、CAPA、投诉、审核）

2. **选择适当的模板：**
- 使用 QM 质量手册模板
- 使用程序模板作为 SOP 的示例
- 根据组织的需求调整结构

3. **使用用户特定信息自定义模板：**
- 替换所有占位符文本：[公司名称]、[日期]、 [姓名]等
- 根据用户的实际操作定制范围
- 根据适用性添加或删除部分
- 确保与组织流程的一致性

4. **关键定制区域：**
- 公司信息和地址
- 产品类型和分类
- 适用的监管要求
- 组织结构和职责
- 实际流程和程序
- 文档编号方案
- 排除和理由

5. **验证完整性：**
- 提供所有必需的部分
- 替换所有占位符
- 交叉引用正确
- 批准部分完成

**文件创建优先顺序：**

**阶段 1 - 基础（关键）：**
1. 质量手册
2. 质量政策和目标
3. 文件控制程序
4. 记录控制程序

**阶段 2 - 核心流程（高优先级）：**
5. 纠正和预防措施 (CAPA)
6. 投诉处理
7. 内部审计
8. 管理评审
9. 风险管理

**阶段 3 - 产品实现（高优先级）优先）：**
10. 设计和开发（如果适用）
11. 采购
12. 生产和服务提供
13. 不合格产品控制

**第 4 阶段 - 支持流程（中优先级）：**
14. 培训和能力
15. M&M 设备的校准/控制
16. 过程验证
17. 产品识别和可追溯性

**第 5 阶段 - 附加要求（中优先级）：**
18. 反馈和上市后监督
19. 监管报告
20. 客户沟通
21. 数据分析

**第 6 阶段 - 专业化（如果适用）：**
22. 安装（如果适用）
23. 维修（如果适用）
24. 灭菌（如果适用）
25. 污染控制（如果适用）

### 4. 制定具体文件

#### 创建质量手册

**流程：**

1. **阅读综合指南：**
- 完整阅读`references/quality-manual-guide.md`
- 了解结构和所需内容
- 查看提供的示例

2. **收集组织信息：**
- 法定公司名称和地址
- 产品类型和分类
- 组织结构
- 适用法规
- 经营范围
- 任何需要排除

3. **使用模板：**
- 以`assets/templates/quality-manual-template.md`开头
- 严格遵循结构（ISO 13485要求）
- 替换所有占位符

4. **完成所需部分：**
- **部分0:** 文档控制、批准
- **第 1 节：** 简介、公司概述
- **第 2 节：** 范围和排除（关键 - 必须证明排除的合理性）
- **第 3 节：** 质量政策（必须由最高管理层签署）
- **第 4-8 节：** 在政策级别解决每个ISO 13485条款
- **附录：** 程序清单、组织结构图、流程图、定义

5. **关键要求：**
- 必须参考所有 31 个记录程序（附录 A）
- 必须描述流程交互（附录 C - 创建流程图）
- 必须定义文档结构（第 4.2 节）
- 必须证明任何排除的合理性（第 2.4 节）

6. **验证检查表：**
- [ ] 所有必需内容根据ISO 13485第 4.2.2 条
- [ ] 由最高管理层签署的质量政策
- [ ] 所有排除均合理
- [ ] 附录 A 中列出的所有程序
- [ ] 包括流程图
- [ ] 包括组织结构图

#### 创建程序（SOP）

**所有程序的一般方法：**

1. **了解要求：**
- 阅读`references/iso-13485-requirements.md`中的相关条款
- 了解必须记录的内容
- 确定您的组织的人员、时间和地点

2. **使用模板结构：**
- 按照CAPA或文档控制模板作为示例
- 标准部分：目的、范围、定义、职责、程序、记录、参考资料
- 保持程序清晰且可操作

3. **明确定义职责：**
- 确定特定角色（不是名称）
- 定义每个角色的职责
- 确保覆盖所有必需的活动

4. **记录“什么”而不是过多的“如何”：**
- 程序应定义必须做什么
- 详细的操作方法包含在工作说明中（第 3 层）
- 在指导和灵活性之间取得平衡

5. **包括必需的元素：**
-ISO 13485条款中指定的所有元素
- 必须维护的记录
- 每项活动的责任
- 相关文件的参考

**示例：创建CAPA过程**

1. 从参考文献
中阅读ISO 13485条款 8.5.2 和 8.5.3 2. 使用`assets/templates/procedures/CAPA-procedure-template.md`
3. 为您的组织自定义：
-CAPA优先级标准
- 您将使用的根本原因分析方法
- 审批机构和责任
- 基于您的运营的时间表
- 与投诉处理、审计等集成
4. 添加表格作为附件：
-CAPA请求表
- 根本原因分析工作表
- 行动计划模板
- 有效性验证清单

#### 创建医疗器械文件 (MDF)

**什么是 MDF：**
- 每个医疗器械类型或系列的文件
- 替换单独的DHF，DMR、DHR（根据FDA QMSR协调）
- 包含有关设备

**所需内容，根据ISO 13485第 4.2.3 条：**

1. 一般说明和预期用途
2. 标签和使用说明规范
3. 产品规范
4. 制造规范
5. 采购、制造、维修程序
6. 测量和监控程序
7. 安装要求（如果适用）
8. 风险管理文件
9. 验证和确认信息
10. 设计和开发文件（如适用）

**流程：**

1. 识别每种器件类型或系列
2. 创建 MDF 结构（文件夹或活页夹）
3. 收集或创建每个所需元素
4. 确保文档之间的可追溯性
5. 维护为动态文档（随更改更新）

### 5. 进行全面差距分析

**何时使用：** 用户希望对所有需求进行详细评估

**流程：**

1. **使用全面检查表：**
- 打开`references/gap-analysis-checklist.md`
- 逐条进行工作
- 标记每个要求的状态：合规、部分、不合规、不适用

2. **对于每个条款：**
- 阅读要求描述
- 识别现有证据
- 注意差距或缺陷
- 定义所需的行动
- 分配责任和目标日期

3. **按条款汇总：**
- 计算每个条款的合规百分比
- 识别最高风险差距
- 确定行动的优先级

4. **创建行动计划：**
- 列出所有差距
- 优先级：关键 > 高 > 中 > 低
- 分配所有者和日期
- 估计所需资源

5. **输出：**
- 已完成的差距分析清单
- 包含合规百分比的摘要报告
- 优先行动计划
- 时间表和里程碑

## 常见场景

### 场景 1：从头开始

**用户请求：** “我们是一家医疗设备初创公司，需要实施ISO 13485。我们从哪里开始？”

**方法：**

1. **解释过程：**
-ISO 13485需要全面的QMS文档
- 通常需要 6-12 个月才能完全实施
- 可以逐步完成

2. **从基础开始：**
- 质量政策和目标
- 质量手册
- 组织结构和职责

3. **遵循优先顺序：**
- 使用上面的阶段 1-6 优先级列表
- 按逻辑顺序创建文档
- 在以前创建的文档基础上构建

4. **关键里程碑：**
- 第 1-2 月：基础文件（质量手册、政策）
- 第 3-4 月：核心流程（CAPA、投诉、审核）
- 第 5-6 月：产品实现流程
- 第 7-8 月：支持流程
- 第 9-10 个月：内部审核和完善
- 第 11-12 个月：管理评审和认证审核

### 场景 2：现有QMS

**用户请求：** “我们有一些程序，但不知道我们缺少什么ZXQ代码4QXZ。”

**方法：**

1. **运行自动差距分析：**
- 询问文档目录
- 运行`scripts/gap_analyzer.py`
- 查看自动结果

2. **进行详细评估：**
- 使用综合评估针对用户特定情况的清单
- 比自动分析更深入
- 评估现有文档的质量，而不仅仅是存在

3. **提供优先差距清单：**
- 缺少强制性程序
- 不完整的程序
- 现有文件的质量问题
- 缺少记录或表格

4. **创建补救计划：**
- 高优先级：安全相关、监管要求
- 中优先级：核心QMS流程
- 低优先级：改进机会

### 场景 3：创建特定文档

**用户请求：** “帮我创建一个CAPA过程。”

**方法：**

1. **解释要求：**
- 从参考文献中阅读ISO 13485第 8.5.2 和 8.5.3 条
- 解释CAPA程序中必须包含哪些内容
- 提供良好的示例CAPA处理

2. **使用模板：**
- 从CAPA过程模板开始
- 解释每个部分的用途
- 显示需要定制的内容

3. **收集用户特定信息：**
- 如何在中启动 CAPA他们的组织？
- 谁是责任方？
- 什么优先级标准有意义？
- 他们将使用什么 RCA 方法？
- 合适的时间范围是多少？

4. **创建自定义程序：**
- 替换所有占位符
- 适应用户流程
- 确保完整性

5. **添加支持材料：**
-CAPA申请表
- RCA 工作表
- 行动计划模板
- 有效性验证清单

### 场景 4：更新监管变更

**用户请求：** “我们需要更新我们的QMS以实现FDA QMSR协调。”

**方法：**

1. **解释更改：**
-FDA21 CFR 第 820 部分与ISO 13485协调一致
- 现在称为 QMSR（2026 年 2 月 2 日生效）
- 关键更改：医疗设备文件替换DHF/DMR/DHR

2. **查看当前文档：**
- 识别引用 QSR
的文档 - 查找单独的DHF、DMR、DHR结构
- 检查ISO 13485合规性差距

3. **更新策略：**
- 将引用从 QSR 更新到 QMSR
- 将DHF/DMR/DHR合并到医疗器械文件中
- 添加任何缺失的内容ISO 13485要求
- 过渡期间保持向后兼容性

4. **创建过渡计划：**
- 更新质量手册
- 更新 MDF 程序
- 重新组织设备历史文件
- 对人员进行变更培训

### 场景 5：准备认证审核

**用户请求：** “我们已准备好文件。我们如何准备认证审核？”

**方法：**

1. **进行准备情况评估：**
- 使用全面的差距分析清单
- 检查所有文档的完整性
- 验证所有必需项目是否存在记录
- 检查实施是否一致

2. **预审核清单：**
- [ ] 所有 31 个程序均已记录并获得批准
- [ ] 质量手册包含所有必需内容
- [ ] 所有产品的医疗器械文件完整
- [ ] 内部审核已完成并已解决问题
- [ ] 已完成管理审核
- [ ] 接受过QMS程序培训的人员
- [ ] 根据保留要求维护的记录
- [ ]CAPA系统正常运行且有效性得到证明
- [ ] 投诉系统可运行

3. **进行模拟审核：**
- 使用ISO 13485要求作为审核标准
- 样本记录以验证一致的实施
- 与人员面谈以验证理解情况
- 识别任何不符合项

4. **解决发现的问题：**
- 纠正任何缺陷
- 文档更正
- 验证有效性

5. **最终准备：**
- 简要介绍管理人员和员工
- 准备审核时间表
- 整理证据和记录
- 指定护送和支持人员

## 最佳实践

### 文件开发

1. **从政策层面开始，然后添加细节：**
- 质量手册 = 政策层面
- 程序 = 内容、人员、时间
- 工作指导 = 详细操作方法
- 表格 = 数据收集

2. **保持一致性：**
-在整个过程中使用相同的术语
- 交叉引用相关文档
- 保持编号方案一致
- 一起更新所有相关文档

3. **为您的受众撰写：**
- 清晰、简单的语言
- 避免行话
- 定义技术术语
- 提供有用的示例

4. **使程序可用：**
- 面向行动的语言
- 逻辑流程
- 明确的职责
- 现实的时间框架

### 排除

**何时可以排除：**
- 设计和开发（如果仅合同制造商）
- 安装（如果产品不需要安装）
- 维修（如果不提供）
- 灭菌（如果非无菌产品）

**合理性要求：**
- 必须在质量手册中
- 必须解释为什么排除
- 不能排除如果执行流程
- 不能影响提供安全、有效设备的能力

**良好理由示例：**
>“不包括第 7.3 条设计和开发。ABC 公司作为合同制造商运营，根据客户提供的完整设计规范生产医疗设备。所有设计活动均由客户执行，ABC 公司对设计输入、输出、验证、确认或设计变更不承担任何责任。”

**糟糕的理由示例：**
>“我们不做设计。” （太简短，无法解释原因或表明没有影响）

### 应避免的常见错误

1. **逐字复制ISO 13485文本**
- 用自己的话书写
- 描述您的流程
- 使其对您的组织可操作

2. **程序过于详细**
- 程序应该稳定
- 过多的细节属于工作指导
- 平衡指导与灵活性

3. **单独创建文档**
- 确保QMS之间的一致性
-交叉引用相关文档
- 基于先前创建的文档

4. **忘记记录**
- 每个程序都应指定记录
- 定义保留要求
- 确保实际维护记录

5. **批准不充分**
- 质量手册必须由最高管理人员签署
- 所有程序必须正确批准的
- 在文件生效之前对员工进行培训

### 重点监管要求

**FDA（美国）：**
- 21 CFR 第 820 部分（现为 QMSR） - 自 2026 年 2 月起与ISO 13485协调一致
- 设备分类确定要求
- 需要企业注册和设备清单

**EU（欧盟）：**
- MDR 2017/745（医疗器械法规）
- IVDR 2017/746（体外诊断法规）
- 技术文件要求
- CE标记要求

**加拿大：**
- 加拿大医疗器械法规 (SOR/98-282)
- 设备分类系统
- 医疗器械企业许可证（MDEL）

**其他地区：**
- 澳大利亚TGA、日本PMDA、中国NMPA等
- 经常要求或认可ISO 13485认证

### 文件保留

**最低保留时间：** 组织定义的医疗器械的使用寿命

**典型保留期：**
- 设计文件：设备寿命+5-10年
- 制造记录：设备的寿命
- 投诉记录：设备寿命+5-10年
- CAPA记录：至少 5-10 年
- 校准记录：设备保存期限+1个校准周期

**始终遵守可能指定更长期限的适用监管要求。**
